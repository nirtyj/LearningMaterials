How to read this

The framing is OpenAI's published five-round loop, but the material generalises. Each chapter follows the same shape: high-level architecture → mid-level architecture → low-level architecture → component deep dives → real production systems → interview follow-ups → references. ASCII diagrams throughout; no hand-waving.

If the interview is imminent, prioritise: Q5 ChatGPT + the three LLM-serving deep dives (chapters 11–14), then Q3 Queue + Lease tokens (7–8), then the streaming pair (1–2), then the webhook cluster (3–6). The two foundation chapters (15–16) are short and worth knowing cold — they appear inside almost every staff-level discussion.

Right rail of every chapter has Related Chapters and a Tooling Universe chip cloud — the latter is where you'll find the connections between papers, products, and primitives. Keyboard: G opens contents, J/K page through chapters.
