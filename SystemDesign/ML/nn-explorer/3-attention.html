3. Attention Mechanisms

The mechanism that lets models focus on what matters.

Prereq: Tab 1 (neurons, features) → Tab 2 (CNN limitations → need for global context)
3.1

Why Attention?

CNNs see small local windows. Attention sees everything in one step.

CNN

Local Vision

Each neuron sees a 3×3 window. Global context requires 10+ layers stacked. Like reading through a tiny window.

ATTENTION

Global Vision

Every element attends to every other in ONE step. Like seeing the entire page and choosing what to focus on.

3.2

Query, Key, Value (Q, K, V)

Three projections — like a database lookup.

Each input token is projected into three vectors via learned weight matrices:
Q (Query) = "What am I looking for?" — the question each element asks.
K (Key) = "What do I contain?" — what each element advertises about itself.
V (Value) = "Here's my actual content." — the information to be retrieved.

Attention(Q, K, V) = softmax(Q · KT / √dk) · V
Dot product Q·K measures similarity. Softmax converts to probabilities. Output = weighted sum of Values.
3.3

Self-Attention

Every element attends to every other in the same sequence.

In self-attention, Q, K, V all come from the SAME input. Each element asks every other "how relevant are you to me?"

Click a patch to see its attention distribution

Where Self-Attention Is Used

  • Every Transformer Layer — GPT, BERT, ViT all use self-attention as their core operation
  • Document Understanding — captures long-range dependencies (e.g., pronoun resolution across paragraphs)
  • Protein Folding — AlphaFold uses self-attention over amino acid sequences to predict 3D structure
  • Code Completion — understanding function context, variable scope, and API patterns across files
  • Music Generation — attending to musical patterns, rhythm, and harmony across a composition
3.4

Cross-Attention

Queries from one source, Keys+Values from another.

Cross-attention connects two different sequences. The decoder generates Queries ("what do I need?"), and the encoder provides Keys and Values ("here's what I know"). This is how image captioning models attend to image features while generating text, or how translation models attend to source language while producing the target.

SOURCE A (DECODER)

Provides Queries

"What do I need to know?"

SOURCE B (ENCODER)

Provides Keys + Values

"Here's what I offer."

CrossAttn(Qdec, Kenc, Venc) = softmax(Qdec · KencT / √d) · Venc
SELF-ATTENTION

Q, K, V from same sequence

Used in: Encoder, ViT, GPT

CROSS-ATTENTION

Q from one source, K+V from another

Used in: Decoder, VLMs, DALL·E

Cross-Attention Applications

  • Image Captioning — BLIP-2 decoder queries attend to image encoder features
  • Text-to-Image — Stable Diffusion uses cross-attention between text embeddings and image features
  • Machine Translation — decoder attends to encoder's source language representation
  • Visual Question Answering — question tokens attend to image features to find answers
  • Text-Conditioned Generation — any model that generates conditioned on text (video, audio, 3D)
3.5

Multi-Head Attention

Multiple attention "lenses" in parallel — each head learns different relationships.

Instead of one attention function, run h parallel heads (12 in ViT-Base), each with own WQ, WK, WV at reduced dimension (768/12=64 per head). Outputs concatenated and projected back.

MultiHead(Q, K, V) = Concat(head1, ..., headh) · WO
where headi = Attn(Q WiQ, K WiK, V WiV)

Each head: 768/12 = 64 dims. 12 outputs (12×64=768) concatenated and linearly projected. Multiple perspectives simultaneously.

Modern Variants

  • Grouped Query Attention (GQA) — shares KV heads across query heads. 2-4x memory savings. Used in LLaMA 2/3, Mistral.
  • Multi-Query Attention (MQA) — single KV head shared by all query heads. Fastest inference. Used in PaLM, Falcon.
  • Sliding Window Attention — each token only attends to a local window. O(n) complexity. Used in Mistral, Longformer.
