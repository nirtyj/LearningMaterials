5.2

ViT Variants

Scaling the same architecture — more layers, wider hidden dim, more heads.

Variant Layers Heads Hidden MLP Params
ViT-Base/16 12 12 768 3,072 86M
ViT-Large/16 24 16 1,024 4,096 307M
ViT-Huge/14 32 16 1,280 5,120 632M

When to Use Which

  • ViT-Base — best for fine-tuning on small datasets (medical imaging, satellite imagery, custom domains)
  • ViT-Large — general-purpose pre-training, good balance of quality and compute
  • ViT-Huge — research and SOTA benchmarks; diminishing returns for most production uses
  • In production: DINOv2 (self-supervised ViT) is increasingly popular for feature extraction — no labeled data needed
5.3

CNN vs ViT

Two paradigms for visual understanding — local convolutions vs global attention.

Aspect CNN ViT
Core Op Convolution (local 3×3) Self-Attention (global)
Receptive Field Local → grows with depth Global from layer 1
Inductive Bias Strong: locality + translation equiv. Weak: learned from data
Position Info Implicit from grid Explicit embeddings (learnable)
Data Needs Works with small datasets Needs 14M+ pretraining
Compute O(n) per layer in spatial O(n²) in sequence length
ViT with enough data matches/beats CNNs. Without enough data, CNNs win — their locality bias is a useful prior.

Hybrid Approaches (Best of Both Worlds)

  • ConvNeXt — a pure CNN modernized with Transformer tricks (larger kernels, LayerNorm, GELU). Competitive with ViT.
  • EfficientViT — efficient ViT variant designed for mobile and edge deployment
  • DINOv2 — self-supervised ViT that learns powerful features without any labels. Excellent for transfer learning.
  • SAM (Segment Anything) — ViT backbone powering universal image segmentation. Works on any image type.
