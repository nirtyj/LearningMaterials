7. Vision-Language & Multimodal Models

Models that see AND speak — bridging vision and language.

Builds on: Tab 5 (ViT encoder) → Tab 6 (LLM decoder) → Tab 3 (cross-attention for bridging)
7.1

VLM Architecture

Image Encoder (ViT) + Projection + LLM = a model that sees and speaks.

Why This Matters

  • Modularity — each component (encoder, projection, LLM) can be swapped, upgraded, or frozen independently
  • Leverage existing models — use a pre-trained ViT and pre-trained LLM; only train the thin projection layer
  • Efficiency — LLaVA’s simple linear projection trains in under 24 GPU-hours, yet produces a competitive VLM
7.2

Key Architectures

Major models bridging vision and language — from contrastive embeddings to native multimodality.

CLIP Contrastive VLM (2021)
OpenAI
Two encoders trained to align matching image-text pairs. Not generative — embeddings for retrieval and zero-shot classification.
Bridge: Contrastive loss aligns two embedding spaces.
Powers: image search (Google Photos, Pinterest), content moderation, zero-shot classification. DALL-E uses CLIP for guidance.
LLaVA Generative VLM (2023)
UW / Microsoft
CLIP ViT + linear projection + LLaMA. Simple projection surprisingly effective.
Bridge: 1–2 linear layers from ViT→LLM dim.
Open-source VLM for research. Fine-tunable on custom vision-language tasks. Spawned LLaVA-1.5, LLaVA-NeXT.
GPT-4V / 4o MLLM (2023)
OpenAI
Multimodal LLM accepting images, text, audio. Handles interleaved inputs natively.
Bridge: Learned adapter / projection layers.
Used in: accessibility (describing images for blind users), document analysis, UI understanding, visual coding assistance.
Gemini MLLM (2023)
Google
Natively multimodal from pre-training. Not bolted-on — multimodality baked in.
Bridge: Native multimodal tokenization.
Claude MLLM (2024)
Anthropic
Multimodal LLM with vision. Analyzes images, charts, documents alongside text.
Bridge: Internal vision processing pipeline.
Used in: chart analysis, document processing, visual reasoning, scientific figure interpretation.
PaliGemma VLM (2024)
Google
SigLIP (vision) + Gemma (LLM). Open-source, strong on OCR and documents.
Bridge: Linear projection SigLIP→Gemma.
Trend: The line between VLM and MLLM is blurring. Models are converging toward natively multimodal architectures trained end-to-end on interleaved image, text, audio, and video data.
7.3

VLM vs MLLM

Vision-Language Models handle two modalities; Multimodal LLMs handle many.

Aspect VLM MLLM
Modalities Vision + Language (2) Vision + Language + Audio + Video + …
Architecture Encoder + Projection + Decoder Unified natively multimodal model
Training Staged: freeze encoder → train proj → finetune End-to-end on interleaved data
Examples CLIP, LLaVA, PaliGemma, BLIP-2 GPT-4o, Gemini, Claude
VLMs are modular — swap out the encoder or LLM independently. MLLMs trade modularity for deeper integration and potentially stronger cross-modal reasoning.
7.4

Three Bridging Strategies

How visual information gets into the language model.

Linear Projection

Simple matrix multiply. Used in LLaVA. Surprisingly effective and fast.
Used in: LLaVA, PaliGemma

Q-Former / Perceiver

Learned query tokens cross-attend to image features. Compresses visual info.
Used in: BLIP-2, Flamingo

Native Multimodal

All modalities tokenized in one unified transformer from scratch.
Used in: Gemini, GPT-4o
Trend: toward native multimodality. But the modular approach (LLaVA-style) remains practical since it leverages existing strong models.

Choosing a Bridge Strategy

  • Linear Projection — fastest to train, easiest to debug, best when you have strong pre-trained components
  • Q-Former — compresses many visual tokens into fewer queries, saving LLM context length; useful for high-res images
  • Native — best performance ceiling, but requires massive data and compute for pre-training from scratch
