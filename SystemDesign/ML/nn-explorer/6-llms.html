6. Large Language Models

Decoder-only transformers predicting the next token — GPT, LLaMA, Claude.

Prereq: Tab 3 (self-attention) → Tab 4 (decoder block, causal masking)
6.1

LLM Pipeline

Given preceding text, predict the next token. That's the entire paradigm.

The entire LLM is just this pipeline repeated autoregressively — append the predicted token, re-run, predict the next, repeat until done.

Why Next-Token Prediction Works

  • Compression is intelligence — to predict the next token well, the model must learn grammar, facts, reasoning, and world knowledge
  • Autoregressive generation — at inference, the predicted token is appended and the process repeats, producing arbitrarily long text
  • Emergent abilities — at sufficient scale, models exhibit chain-of-thought reasoning, in-context learning, and tool use without explicit training
6.2

Training & Scale

Two-phase training: learn language, then learn to be helpful.

Pre-training

Next-token prediction on trillions of tokens. Learns language, facts, reasoning. Extremely expensive ($millions). The model sees the entire internet-scale corpus and learns to compress it into its weights.

Fine-tuning & RLHF

SFT on instruction-response pairs, then RLHF/DPO for alignment. Makes model helpful and harmless. Much cheaper. Transforms a text completer into an assistant that follows instructions.

Scaling Laws (Chinchilla)

  • Optimal ratio: ~20 tokens per parameter for compute-optimal training
  • LLaMA-3: trained on 15T tokens for 405B params (37x over-trained) — more data beats bigger models
  • Key insight: For a fixed compute budget, it's better to train a smaller model on more data than a larger model on less data

Training Infrastructure

  • Hardware: 1000s of GPUs (H100/A100), distributed across data centers
  • Cost: $10M–$100M+ per training run for frontier models
  • Techniques: Data parallelism, tensor parallelism, pipeline parallelism, ZeRO optimization
Trend: The cost of training frontier models is doubling roughly every 6–9 months. Open-weight models (LLaMA, Mistral) are closing the gap with proprietary models, democratizing access.
6.3

Key Concepts

Six ideas that make modern LLMs practical.

KV Cache

Cache K,V from previous tokens during generation. Avoids recomputation → fast inference. Without KV cache, generation is O(n²) per token. With it, O(n). Essential for production deployment.

Context Window

Max sequence length. Modern: 128K–1M+ tokens. Used for RAG (retrieval augmented generation), long document analysis, multi-turn conversations.

Temperature

Scales logits before softmax. T<1=focused, T>1=creative. Production: T=0 for deterministic outputs (code, math). T=0.7–1.0 for creative tasks.

BPE Tokenization

Iteratively merge frequent character pairs. Balances vocab size and coverage. GPT uses ~50K tokens. Multilingual models (mT5) use ~250K. Bigger vocab = fewer tokens per text but larger embedding table.

MoE (Mixture of Experts)

N expert MLPs + router. Only top-k active per token. More params, same compute. Mixtral 8x7B: 47B total params but only 13B active per token. Best quality/cost ratio.

RoPE

Rotary Position Embedding: encode positions as rotations. Captures relative distance. Enables context extension via interpolation (YaRN, NTK-aware scaling). LLaMA-3: 128K context.

Inference Optimization Stack

  • Quantization — reduce weights from FP16 to INT8/INT4. 2–4x memory savings with minimal quality loss (GPTQ, AWQ, GGUF)
  • Speculative Decoding — small draft model proposes tokens, large model verifies in parallel. 2–3x speedup.
  • Continuous Batching — vLLM/TGI dynamically batch requests, maximizing GPU utilization. Essential for serving at scale.
  • Flash Attention — IO-aware attention algorithm. 2–4x faster, uses far less GPU memory. Now standard in all frameworks.

Prompting & Reasoning Techniques

  • Chain-of-Thought (CoT) — "Let's think step by step" unlocks multi-step reasoning in large models
  • Few-Shot Learning — provide examples in the prompt; no gradient updates needed. Emergent at ~100B+ scale.
  • RAG (Retrieval-Augmented Generation) — retrieve relevant documents, inject into context. Reduces hallucination and grounds responses in facts.
  • Tool Use / Function Calling — models call external APIs (search, code execution, databases) to extend their capabilities beyond text.
