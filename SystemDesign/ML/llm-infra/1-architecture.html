1. Modern LLM Architecture

Decoder-only transformers, attention mechanisms, positional encodings, and the building blocks of frontier models

1.1

Transformer Backbone

The decoder-only architecture that powers every modern LLM

The original 2017 "Attention Is All You Need" paper introduced the Transformer with an encoder-decoder structure designed for sequence-to-sequence tasks like machine translation. Since then, the field has converged on a simpler variant: the decoder-only architecture. GPT, Llama, Mistral, Claude, Gemini, and nearly every frontier LLM uses this paradigm. The core insight is that a single autoregressive decoder, trained with next-token prediction, is a universal learner capable of in-context learning, chain-of-thought reasoning, and instruction following — all without a separate encoder stage.

Three Transformer Families
  • Encoder-only (BERT, RoBERTa): Bidirectional attention over the full input. Ideal for classification and embedding tasks, but cannot generate text autoregressively.
  • Encoder-decoder (T5, BART, Flan-T5): Encoder processes the input bidirectionally; decoder generates output causally. Natural for translation and summarization, but adds architectural complexity.
  • Decoder-only (GPT, Llama, Mistral): A single stack of transformer blocks with causal masking. Scales cleanly, supports generation natively, and dominates the LLM landscape.

In the decoder-only design, each transformer block consists of two sub-layers: masked multi-head self-attention and a position-wise feed-forward network (FFN). Both sub-layers are wrapped in residual connections and normalization. The causal (autoregressive) mask ensures that when predicting token t, the model can only attend to tokens at positions 1, 2, …, t−1. This is implemented as an upper-triangular mask that sets future positions to −∞ before the softmax, ensuring zero attention weight on tokens that haven't been generated yet.

Scaled Dot-Product Attention

Attention(Q, K, V) = softmax( QKᵀ / √d_k ) · V

Q (queries), K (keys), and V (values) are linear projections of the input. The scaling factor √d_k prevents the dot products from growing too large in magnitude, which would push softmax into regions with vanishingly small gradients. This simple formula is the computational heart of every transformer.

Input Token IDs ↓ [Token Embedding + Positional Encoding] ↓ ┌───────────────────────────────────────┐ │ Masked Multi-Head Self-Attention │ ← Causal mask applied here │ + Residual + Norm │ ├───────────────────────────────────────┤ │ Feed-Forward Network (FFN) │ │ + Residual + Norm │ └───────────────────────────────────────┘ ↓ (× N layers) [Linear + Softmax → next-token probabilities]

Residual connections are critical at scale. Without them, gradients would vanish across 80+ layers. The residual stream acts as a "highway" that each sub-layer reads from and writes to, enabling deep networks to train stably.

1.2

Attention Variants

MHA, MQA, and GQA — trading expressivity for inference efficiency

The original transformer uses Multi-Head Attention (MHA), where the input is projected into independent Q, K, and V matrices for each attention head. With h heads and a model dimension d, each head operates on vectors of size d/h. MHA provides maximum expressivity — each head can learn to attend to different relationship patterns — but during autoregressive inference, every head requires its own key and value cache. For a model with 32 heads, this means 32 independent K and V tensors stored per layer, consuming significant GPU HBM.

MQA

Multi-Query Attention

Introduced by Noam Shazeer (2019), MQA uses a single shared K and V projection across all query heads. This reduces the KV cache by a factor of h (e.g., 32× for a 32-head model). The quality degradation is surprisingly small, and the inference throughput improvement is dramatic. Adopted by PaLM, Falcon, and StarCoder.

GQA

Grouped Query Attention

GQA (Ainslie et al., 2023) is the pragmatic middle ground. Query heads are divided into g groups, and each group shares one K and one V head. With g=8 groups for 32 query heads, you get 8 KV heads — a 4× KV cache reduction versus MHA while retaining more expressivity than MQA. This has become the default choice for Llama 2/3, Mistral, Gemma 2, and Qwen 2.5.

Variant KV Heads KV Cache Memory Throughput Adopted By
MHA h (e.g., 32) Baseline (1×) Baseline GPT-3, BERT, original Transformer
MQA 1 1/h (~3%) 2–4× PaLM, PaLM 2, Falcon, StarCoder
GQA g (e.g., 8) g/h (~25%) 1.5–2.5× Llama 2/3, Mistral, Gemma, Qwen 2.5

The GQA paper showed that an MHA model can be "uptrained" to GQA by mean-pooling the existing KV heads within each group, then continuing training for a small fraction of the original compute. This makes GQA a practical retrofit, not just a from-scratch decision.

Multi-Head Latent Attention (MLA)

DeepSeek-V2 and V3 introduce MLA, which goes further than GQA. Instead of separate KV heads, MLA compresses the KV representation into a low-rank latent vector, then expands it back during attention. This achieves even smaller KV caches than GQA while maintaining MHA-level quality. It represents the next frontier in attention efficiency.

1.3

Positional Encoding

Injecting sequence order into a permutation-invariant mechanism

Self-attention is fundamentally permutation-invariant: if you shuffle the input tokens, the attention outputs (before masking) are simply a permuted version of the originals. This means a transformer with no positional information would treat "the cat sat on the mat" identically to "mat the on sat cat the." Positional encoding solves this by injecting information about each token's position in the sequence.

LEGACY

Learned Absolute Embeddings (GPT-2)

The simplest approach: add a learned embedding vector for each position (0, 1, 2, …, max_len). GPT-2 used this with a 1024-token context window. Advantages: trivially simple to implement. Disadvantages: the model cannot extrapolate beyond the trained context length, and the embeddings must be stored as part of the model parameters. No modern frontier model uses this approach.

STANDARD

RoPE — Rotary Position Embeddings

RoPE (Su et al., 2021) has become the near-universal positional encoding for 2024+ LLMs. Rather than adding position information to token embeddings, RoPE encodes relative position directly in the attention computation by rotating Q and K vectors in a complex number space. Each dimension pair (2i, 2i+1) of the Q and K vectors is treated as a 2D vector and rotated by an angle proportional to the position.

R(θ, m) · q = [ q_2i cos(mθ_i) − q_{2i+1} sin(mθ_i) , q_2i sin(mθ_i) + q_{2i+1} cos(mθ_i) ]

The key property is that the dot product between a rotated Q at position m and a rotated K at position n depends only on the relative distance (m−n), giving the model natural relative position awareness. RoPE also supports length extrapolation through techniques like NTK-aware scaling, YaRN, and dynamic NTK, allowing models trained on 8K contexts to generalize to 128K+ at inference time.

ALTERNATIVE

ALiBi — Attention with Linear Biases

ALiBi (Press et al., 2022) takes a radically simple approach: instead of modifying embeddings, it adds a linear bias to the attention score based on the distance between query and key positions. The bias is −m · |i − j|, where m is a head-specific slope (set geometrically, not learned). This acts as a soft local attention window that naturally decays with distance. ALiBi requires no additional parameters and shows strong length extrapolation. Used by MPT and BLOOM, though RoPE has since overtaken it in popularity.

Length Extrapolation Techniques

Training with long sequences is expensive (attention is O(N²)). Length extrapolation methods allow a model trained on short contexts to perform well on longer ones:

  • Position Interpolation (PI): Linearly scales positions down to fit within the trained range. Simple but requires fine-tuning.
  • NTK-aware Scaling: Modifies the RoPE frequency base to handle higher frequencies better. Works with minimal or no fine-tuning.
  • YaRN: Combines NTK scaling with attention temperature adjustments and a ramp function. State-of-the-art extrapolation for RoPE models.
  • Dynamic NTK: Adjusts the scaling factor based on the actual sequence length at inference, applying more interpolation only when needed.
1.4

Activations & Normalization

SwiGLU, RMSNorm, and the pre-norm transformer recipe

The feed-forward network (FFN) in each transformer block is the primary "compute" layer, where the model performs non-linear transformations on each token's representation. While the original transformer used a simple two-layer MLP with ReLU activation, modern LLMs have converged on SwiGLU — a gated linear unit with the Swish activation function. Similarly, normalization has evolved from LayerNorm to the simpler and faster RMSNorm, applied in a pre-norm configuration.

SwiGLU Activation

SwiGLU (Shazeer, 2020) replaces the standard FFN with a gated architecture. Instead of a single linear projection followed by an activation, it uses two parallel projections and combines them with element-wise multiplication (gating):

SwiGLU(x) = Swish(x · W_1 + b_1) ⊙ (x · W_2 + b_2)
Swish(x) = x · σ(βx)    where σ is the sigmoid function

The gating mechanism allows the network to learn which features to pass through, acting as a form of learned feature selection. Empirically, SwiGLU achieves better loss for the same compute budget compared to ReLU or GELU FFNs. Note that SwiGLU requires three weight matrices (W_1, W_2, W_3) instead of two, so the hidden dimension is typically reduced from 4×d to (8/3)×d to keep the parameter count constant. Used by Llama, Mistral, Gemma, Qwen, PaLM, and virtually all frontier models.

NORMALIZATION

RMSNorm

RMSNorm(x) = x / RMS(x) · γ
RMS(x) = √( (1/n) ∑ x_i² )

RMSNorm (Zhang & Sennrich, 2019) simplifies LayerNorm by removing the mean-centering step. It normalizes only by the root mean square, then applies a learned scale parameter γ. This removes one reduction operation and is empirically equivalent in quality while being ~10-15% faster. Adopted by Llama, Mistral, Gemma, and most modern LLMs.

PLACEMENT

Pre-Norm vs Post-Norm

The original transformer applies normalization after the attention/FFN sub-layer (post-norm). Modern LLMs universally use pre-norm: normalize the input before passing it to attention/FFN. Pre-norm provides significantly better training stability, especially for deep models (60+ layers), because the residual connection carries unnormalized activations, preventing gradient explosion. The trade-off is that pre-norm can slightly reduce representation power, but this is negligible at scale.

Pre-Norm Transformer Block: x → [RMSNorm] → [Multi-Head Attention] → (+x) → out_1 residual out_1 → [RMSNorm] → [SwiGLU FFN] → (+out_1) → out_2 residual

The combination of Pre-Norm + RMSNorm + SwiGLU has become the "standard recipe" for modern LLM blocks. It first appeared in PaLM (2022), was adopted by Llama (2023), and is now essentially universal in open-weight models.

1.5

FlashAttention

IO-aware exact attention — the algorithm that unlocked long contexts

Standard attention computes the full N×N attention matrix, where N is the sequence length. This matrix must be materialized in GPU HBM (high-bandwidth memory), consuming O(N²) memory. For a 128K-token sequence, this means storing a matrix with ~16 billion entries — clearly impractical. Even for shorter sequences, the memory reads and writes between HBM and SRAM (on-chip cache) dominate the runtime, making attention memory-bandwidth-bound rather than compute-bound.

The FlashAttention Algorithm (Dao et al., 2022)

FlashAttention is an IO-aware algorithm that restructures the attention computation to minimize data movement between HBM and SRAM. The key ideas:

  1. Tiling: Divide Q, K, V into blocks that fit in SRAM (the fast on-chip memory, ~20 MB on A100).
  2. Block-wise computation: Compute attention for each tile pair in SRAM without ever writing the full N×N matrix to HBM.
  3. Online softmax: Use the "online softmax trick" to incrementally compute the correct softmax across tiles, tracking running max and sum statistics.
  4. Recomputation on backward pass: Instead of storing the attention matrix for the backward pass, recompute it from Q, K, V blocks. This trades FLOPs for memory.

The result is exact attention (not an approximation) with O(N) memory instead of O(N²), and 2-4× wall-clock speedup due to reduced HBM traffic. FlashAttention is now the default in virtually every training and inference framework.

GPU Memory Hierarchy

Understanding FlashAttention requires knowing the GPU memory hierarchy. On an A100: HBM is 80 GB at ~2 TB/s bandwidth; SRAM is ~20 MB at ~19 TB/s bandwidth. SRAM is 10× faster but 4000× smaller. FlashAttention keeps the hot data (the current tile of Q, K, V) in SRAM and only reads/writes final results to HBM.

Version Year Key Improvement Target Hardware
FlashAttention 2022 IO-aware tiling, online softmax, O(N) memory A100 (FP16/BF16)
FlashAttention-2 2023 Better parallelism across sequence dim, reduced non-matmul FLOPs, ~2× over FA-1 A100, H100
FlashAttention-3 2024 FP8 tensor core support, asynchronous softmax, warp-specialized pipelining H100 (Hopper architecture)

FlashAttention didn't change the mathematics of attention at all — it changed the memory access pattern. This is a powerful lesson: at modern hardware scales, algorithmic efficiency is often about IO, not FLOPs.

1.6

Mixture-of-Experts (MoE)

Scaling total parameters while keeping per-token compute constant

In a standard dense transformer, every parameter is active for every token. If a model has 70 billion parameters, all 70B are used to process each token. This creates a tight coupling between model capacity (total parameters) and compute cost (FLOPs per token). Mixture-of-Experts (MoE) breaks this coupling by replacing the single FFN in each transformer block with N parallel "expert" FFN modules and a learned router (gating network) that selects the top-k experts for each token.

How MoE Routing Works

For each token, the router computes a probability distribution over all N experts using a softmax over learned gate weights. The top-k experts (typically k=2) are selected, and the token is processed by only those experts. The outputs are combined as a weighted sum using the router probabilities:

output = ∑_{i ∈ top-k} g_i · Expert_i(x)    where g_i = softmax(x · W_gate)_i

This means a model with 8 experts and top-2 routing activates only ~25% of its FFN parameters per token. Mixtral 8×7B has ~47B total parameters but only ~13B active per token, achieving performance competitive with dense 70B models at a fraction of the inference cost.

N│ │ (FFN) │ │ (FFN) │ │ (FFN) │ │ (FFN) │ └──────┘ └──────┘ └──────┘ └──────┘ │* │ │ └──────────┐ ┌──────────┘ │ │ v v ┌───────────────────────┐ │ Weighted Sum of │ │ top-k Expert Outputs │ * selected by router └───────────────────────┘
MoE Challenges
  • Load balancing: Without intervention, the router may collapse to using only a few experts, wasting capacity. An auxiliary load-balancing loss penalizes uneven expert utilization.
  • Expert parallelism: Experts must be distributed across GPUs, requiring all-to-all communication to route tokens to their selected experts and gather results back. This is a significant networking bottleneck.
  • Memory footprint: While active parameters are sparse, all expert weights must still be loaded in memory. Mixtral 8×7B needs ~47B params in VRAM, even though only ~13B are active per token.
  • Training instability: Router training can be unstable, with experts alternating between being over- and under-utilized. Techniques like expert choice routing and soft routing help stabilize training.

Notable MoE Models

  • Mixtral 8×7B — 8 experts, top-2, ~47B total / ~13B active. Matches Llama 2 70B on many benchmarks.
  • GPT-4 — Rumored MoE with ~1.8T total parameters, 8 experts per layer.
  • DeepSeek-V2/V3 — 671B total / 37B active. Uses fine-grained experts (256 experts, top-8) with shared experts always active.
  • DBRX — Databricks' 132B MoE (36B active), 16 experts with top-4 routing.
  • Qwen2-MoE — 57B total / 14B active. Fine-grained expert segmentation with shared experts.

The trend is moving toward fine-grained MoE: more experts with smaller individual sizes and higher top-k selection. DeepSeek-V3 uses 256 routed experts + 1 shared expert, selecting top-8. This gives more diverse expert combinations and better load balancing.

Model Landscape

A comparative view of the major LLM families and their architectural choices

The LLM landscape has consolidated around a shared set of architectural primitives — decoder-only transformers, RoPE, GQA, SwiGLU, RMSNorm, pre-norm, FlashAttention — but each model family makes distinct choices about scale, training data, context length, and specializations like MoE or multimodality. Key Architectural Trends (2024-2025)
  • GQA is the default. Almost every new open model uses grouped-query attention over full MHA.
  • RoPE is universal. ALiBi and learned absolute embeddings have been fully displaced.
  • MoE is mainstream. After Mixtral proved the concept, DeepSeek-V3 and Qwen2-MoE showed that fine-grained MoE can match dense models at a fraction of the compute.
  • Context windows are exploding. 128K is standard; Gemini supports 1M+; research pushes toward 10M tokens.
  • Multimodality is table stakes. Vision, audio, and video encoders are being fused into the same transformer backbone.
  • Open weights accelerate. Meta's Llama, Google's Gemma, Alibaba's Qwen, and DeepSeek have created a thriving open ecosystem.
Emerging: Multi-Head Latent Attention (MLA) from DeepSeek represents a potential next step beyond GQA, compressing KV representations into low-rank latents. Native FP8 training (DeepSeek-V3) and dynamic expert allocation are also areas of active innovation.

While proprietary models (GPT-4, Claude, Gemini) don't disclose architectures, the open-weight ecosystem — Llama, Mistral, Gemma, Qwen, DeepSeek — has converged on a remarkably consistent recipe: decoder-only + GQA + RoPE + SwiGLU + RMSNorm + Pre-Norm + FlashAttention. The differentiators are increasingly about data, scale, alignment, and training infrastructure rather than novel architecture.
