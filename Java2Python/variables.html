Variables & Memory Model

References, primitives, and the mutability that bites you

🐍 Python
# Everything is an object reference
x = 42          # x references int object
y = x           # y references SAME object
x = 100         # x now references NEW object

# No type declarations needed
name = "Alice"  # str
age = 25        # int
pi = 3.14       # float
is_valid = True # bool

# Dynamic typing
x = 10          # int
x = "hello"     # now str (allowed!)
☕ Java
// Primitives stored directly on stack
int x = 42;       // value stored directly
int y = x;        // COPY of value
x = 100;          // only x changes

// Type declarations required
String name = "Alice";
int age = 25;
double pi = 3.14;
boolean isValid = true;

// Static typing
int x = 10;
// x = "hello";  // ERROR! Type mismatch
Python Memory

All variables are references to heap objects

x → int(42) @ 0x7f...
name → str("Alice") @ 0x8a...
nums → list([1,2,3]) @ 0x9b...
Java Memory

Primitives on stack, objects on heap

x = 42 (stack - primitive)
name → String @ heap
nums → ArrayList @ heap

💡 Interview Insight: Mutability

  • Python: Lists are mutable, tuples/strings are immutable
  • Java: Arrays are mutable, Strings are immutable
  • In Python, a = b for lists creates a reference, not a copy!
  • Use a = b.copy() or a = b[:] for shallow copy
