No temp variable needed!

arr[start:end:step] arr[-1] # last element arr[:-1] # all but last arr[::-1] # reversed

📊 DefaultDict

from collections import defaultdict d = defaultdict(list) d = defaultdict(int) # counters

🎚️ Compact Sort & Search Idioms

Four small idioms that collapse 5–10 lines of Java logic into one Python expression. Show up constantly in Reconstruct Queue, Largest Number, Russian Doll Envelopes, Find K Closest Elements, Top K Frequent. 

# ── 1) Multi-key sort with mixed direction ──
# Tuple key sorts lexicographically. Use - on numerics to flip direction.
people.sort(key=lambda p: (-p[0], p[1]))   # height desc, k asc (LC 406)
envelopes.sort(key=lambda e: (e[0], -e[1])) # LC 354: width asc, height desc
# For non-numeric reverse keys, use cmp_to_key (already covered above).

# ── 2) next((x for x in arr if cond), default) — find-first ──
# Replaces the for/return/else dance.
first_even = next((x for x in nums if x % 2 == 0), -1)

# With index — pair enumerate + next:
i = next((i for i, x in enumerate(nums) if x > target), len(nums))

# ⚠️ Without the default, StopIteration leaks out — always supply one.

# ── 3) max(d, key=d.get) — argmax of a dict ──
# Returns the KEY whose VALUE is largest. (max(d) alone returns max key.)
freq = Counter(nums)
mode = max(freq, key=freq.get)        # most-frequent element

# Same with min, and arbitrary key fns:
furthest = max(points, key=lambda p: p[0]**2 + p[1]**2)

# ── 4) heapq.heappushpop / heapreplace — atomic top-K ──
# heappushpop(h, x): push then pop, ONE sift-down (faster than push + pop).
# heapreplace(h, x): pop then push — same cost. Use when h is non-empty.
import heapq
def topK(nums, k):
    h = nums[:k]
    heapq.heapify(h)                    # O(k) min-heap of first k
    for n in nums[k:]:
        if n > h[0]:
            heapq.heapreplace(h, n)     # ← one sift, not two
    return h                            # the top k (unsorted)

# ── Bonus: deque(maxlen=k) — auto-evicting fixed window ──
from collections import deque
window = deque(maxlen=k)                # appending past k auto-pops left
for x in stream:
    window.append(x)                    # window holds at most last k

💡 Why each one

  • Multi-key sort: Java's Comparator.comparing(...).thenComparing(...).reversed() chains collapse to a 1-line tuple key. Negation only works on numerics — for strings, use two passes (Python's sort is stable) or cmp_to_key
  • next(gen, default): O(1) memory, short-circuits on first match. Forgetting the default is the #1 bug — always supply one
  • max(d, key=d.get): argmax in 4 tokens. Ties broken by insertion order (3.7+) — call sorted(d, key=...) if you need a deterministic tiebreak
  • heapreplace vs heappush + heappop: same big-O but ~2× faster in practice — important inside top-K hot loops on large streams
