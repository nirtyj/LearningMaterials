📋 List / Array

Python list vs Java ArrayList & primitive arrays

Comparison

📋 Dynamic Array

Python list = [1, 2, 3]
Java ArrayList<Integer> list = new ArrayList<>();

Python lists can hold mixed types; Java requires generics

📊 Fixed Array

Python arr = [0] * 10 # or use array module
Java int[] arr = new int[10];

Java primitive arrays are more memory efficient

Initialization

🐍 Python
nums = []                          # empty list
nums = [1, 2, 3]                  # literal
nums = [0] * 10                   # 10 zeros
nums = list(range(5))            # [0,1,2,3,4]
nums = [x**2 for x in range(5)]  # [0,1,4,9,16]

# ── 2D List (Matrix) ──
grid = [[0] * cols for _ in range(rows)]
# ⚠️ WRONG: [[0]*cols]*rows (shared refs!)
☕ Java
List<Integer> nums = new ArrayList<>();
List<Integer> nums = new ArrayList<>(Arrays.asList(1,2,3));
List<Integer> nums = List.of(1,2,3); // immutable!
int[] arr = new int[10];           // 10 zeros
// No built-in range → use IntStream

// ── 2D Array ──
int[][] grid = new int[rows][cols];
// Or with ArrayList:
// List<List<Integer>> grid = new ArrayList<>();

⚠️ 2D Grid Gotcha

  • [[0]*cols]*rows creates rows references to the SAME inner list — mutating one row mutates them all
  • Always use a comprehension: [[0]*cols for _ in range(rows)]

📋 Copy & extend nuances

# ── Append vs allocate-new ──
a.extend(b)                      # mutate in place (same as: a += b)
a += b                           # same as extend on a list
c = a + b                        # NEW list; a, b unchanged — use when you must preserve a

# ── Three shallow-copy idioms (all equivalent) ──
b = a[:]                         # slice copy — terse, very common
b = a.copy()                     # method form (3.3+) — most readable
b = list(a)                      # works for any iterable, not just lists

# ── Deep copy for nested structures ──
import copy
b = copy.deepcopy(a)             # required when a contains lists/dicts/objects

# ── Reverse: in-place vs new ──
a.reverse()                      # mutates a (returns None — don't chain!)
b = a[::-1]                      # NEW reversed list — leaves a alone
for x in reversed(a): ...        # iterator only, no copy — fastest if you only iterate

For why a = b isn't a copy at all (and bites you in interviews), see Common Gotchas → List Copy vs Reference.

🛠️ Built-ins commonly paired with lists

When you write list code in interviews, these are the built-ins you reach for most.

# ── Iterating ──
for v in arr: ...
for i, v in enumerate(arr): ...           # index + value
for v in reversed(arr): ...               # iterator, no copy
for a, b in zip(arr, arr[1:]): ...        # adjacent pairs

# ── Searching ──
target in arr                              # O(n) membership
arr.index(x)                              # first index of x (raises if missing)
next((i for i, v in enumerate(arr) if v > k), -1)  # find-first index, safe default

# ── Sorting ──
sorted(arr, key=lambda x: -x)              # custom key, NEW list
sorted(arr, key=lambda x: (-x[0], x[1]))   # multi-key
arr.sort()                                # in place, returns None — don't chain!

# ── Aggregating ──
sum(arr), min(arr), max(arr)
sum(arr, start=10)                        # offset
any(x > 0 for x in arr)
all(x > 0 for x in arr)

# ── Transforming ──
[x*x for x in arr]                        # comprehension — list (materialized)
(x*x for x in arr)                        # generator — lazy, one value at a time
list(map(str, arr)), list(filter(None, arr))

Operations

🐍 Python
nums = [3, 1, 4, 1, 5]

# ── Add ──
nums.append(9)              # [3,1,4,1,5,9]
nums.insert(0, 7)           # [7,3,1,4,1,5,9]
nums.extend([2, 6])         # append multiple

# ── Access ──
nums[0]                     # first → 7
nums[-1]                    # last → 6
nums[1:4]                   # slice → [3,1,4]

# ── Remove ──
nums.pop()                  # remove last → 6
nums.pop(0)                 # remove first → 7
nums.remove(4)              # remove first occurrence of 4

# ── Search ──
1 in nums                  # True (O(n))
nums.index(1)               # first index of 1
nums.count(1)               # how many 1s

# ── Sort ──
nums.sort()                 # in-place ascending
nums.sort(reverse=True)     # in-place descending
sorted(nums)                # new sorted list
sorted(nums, key=lambda x: -x)  # custom sort

# ── Reverse ──
nums.reverse()              # in-place
nums[::-1]                  # new reversed list

# ── Size ──
len(nums)                   # length

# ── Comprehension (Python superpower!) ──
squares = [x**2 for x in range(10)]
evens = [x for x in nums if x % 2 == 0]
flat = [x for row in grid for x in row]
☕ Java
List<Integer> nums = new ArrayList<>(
    Arrays.asList(3, 1, 4, 1, 5));

// ── Add ──
nums.add(9);                // [3,1,4,1,5,9]
nums.add(0, 7);             // [7,3,1,4,1,5,9]
nums.addAll(Arrays.asList(2, 6)); // append multiple

// ── Access ──
nums.get(0);                // first → 7
nums.get(nums.size()-1);    // last → 6
nums.subList(1, 4);         // slice → [3,1,4]

// ── Remove ──
nums.remove(nums.size()-1); // remove last
nums.remove(0);             // remove at index 0
nums.remove(Integer.valueOf(4)); // remove value 4

// ── Search ──
nums.contains(1);           // true (O(n))
nums.indexOf(1);            // first index of 1
Collections.frequency(nums, 1); // count of 1

// ── Sort ──
Collections.sort(nums);     // in-place ascending
Collections.sort(nums, Collections.reverseOrder());
nums.stream().sorted().collect(Collectors.toList());
nums.sort((a,b) -> b - a);  // custom sort

// ── Reverse ──
Collections.reverse(nums);  // in-place
// No one-liner for new reversed list

// ── Size ──
nums.size();                // length

// ── Stream API (closest to comprehension) ──
int[] squares = IntStream.range(0,10)
    .map(x -> x*x).toArray();
// nums.stream().filter(x -> x%2==0).toList();

⏪ Reverse Iteration Idioms — pick the right form

Several Python forms iterate "backward" but they're not interchangeable. Choose by what you need: indices vs values, full vs sliced, custom step. 

# 1) Walk indices last → first  (when you need i, not arr[i] alone)
for i in range(len(arr) - 1, -1, -1):
    ...   # i = 5,4,3,2,1,0  for arr of length 6

# 2) Walk values last → first  (when index doesn't matter)
for v in reversed(arr):
    ...

# 3) Walk a slice in reverse  (skip arr[0], or any sub-range)
for v in reversed(arr[1:]):
    ...   # yields arr[-1], arr[-2], ..., arr[1]

# 4) Same as (1), more explicit about "reverse of forward range"
for i in reversed(range(len(arr))):
    ...

# 5) New reversed list (allocates a copy — usually overkill)
for v in arr[::-1]:
    ...

# 6) Backward with custom step (only range() supports this)
for i in range(len(arr) - 1, -1, -2):
    ...   # every other index, last → first

📐 Why range(len(arr)-1, -1, -1) uses -1 as stop

  • start = len(arr) - 1 → last valid index
  • stop = -1 → "one past 0 going backward" — because stop is exclusive
  • step = -1 → count downward
  • ⚠️ range(len(arr)-1, 0, -1) is the common bug — it stops at 1 and misses index 0

🎯 When to use which

  • Need the index (e.g. writing back to arr[i], two-pointer math) → form (1) or (4)
  • Need the value only → form (2); it's an iterator (no copy) and reads cleanest
  • Need to skip a prefix/suffix in reverse → form (3): reversed(arr[1:]), reversed(arr[:-1]), etc. (slice allocates a small copy)
  • Need a non-default step (every other, every third) → form (6); reversed() can't express this
  • Avoid form (5) arr[::-1] in loops — it allocates a full reversed copy

🧮 Matrix & Index Tricks — zip(*m), divmod, ~i

Three idioms that compress matrix rotation, grid problems, and palindrome two-pointer code from many lines to one. Heavily used in Rotate Image, Transpose Matrix, Word Search, Spiral Matrix, Valid Palindrome. 

# ── 1) zip(*matrix) — transpose / iterate columns ──
matrix = [[1,2,3], [4,5,6], [7,8,9]]
list(zip(*matrix))             # [(1,4,7),(2,5,8),(3,6,9)]
# Rotate 90° clockwise (LC 48): transpose then reverse each row
rotated = [list(reversed(row)) for row in zip(*matrix)]

# Iterate column-by-column — no nested index gymnastics
for col in zip(*matrix):
    print(sum(col))             # sum each column

# ── 2) divmod(i, cols) — flat index ↔ (row, col) ──
# Lets you use a single int set/dict for visited (faster hashing
# than tuple keys) on rows × cols grids.
for i in range(rows * cols):
    r, c = divmod(i, cols)         # i=7, cols=3 → r=2, c=1
    # grid[r][c] ...

# Inverse: flat = r * cols + c

# ── 3) arr[~i] — i-th from the end (~i == -i-1) ──
# arr[~0] == arr[-1], arr[~1] == arr[-2], ...
# Palindrome check without len() arithmetic:
for i in range(len(s) // 2):
    if s[i] != s[~i]:                # ← i and ~i meet in the middle
        return False

# Rotate-image inner swap (LC 48): cleaner than matrix[n-1-j][i]
matrix[i][j], matrix[~j][i], matrix[~i][~j], matrix[j][~i] = \
    matrix[~j][i], matrix[~i][~j], matrix[j][~i], matrix[i][j]

💡 Why these matter

  • zip(*matrix) works because * unpacks the rows as positional args to zip — Python's transpose, no NumPy needed
  • divmod + flat indexing is the standard speedup for BFS/DFS on grids when you want a set[int] visited instead of set[tuple]
  • ~i reads as "complement of i" — for any non-negative i, ~i equals -i-1, which Python interprets as "i-th from the end". Replaces every n - 1 - i in your code
