📐 Syntax Templates

Three multi-slot templates (comprehension, slice, range) and three list-mutation patterns (copy, deepcopy, append/extend) — labeled syntax plus concrete input → output so you can predict the result before running it.

📐 List comprehension template

Memorize the four named slots. The filter is optional.

[ what_to_return  for thing in things  if keep_condition ]
        ↑                  ↑                    ↑
    expression         iteration         optional filter

# Example 1: no filter — square every number
[x*x for x in range(5)]
# → [0, 1, 4, 9, 16]

# Example 2: with filter — keep only even numbers
nums = [1, 2, 3, 4, 5, 6]
[x for x in nums if x % 2 == 0]
# → [2, 4, 6]

# Example 3: transform AND filter — squares of evens
[x*x for x in nums if x % 2 == 0]
# → [4, 16, 36]
  • what_to_return runs once per item that passes the filter — it's the value placed into the new list
  • thing in things is the same as a for loop header — thing is the loop variable
  • if keep_condition is optional; drop it when you want every item
  • Swap the outer [] for () to get a generator (lazy, one value at a time) — see the Built-ins box below

🔪 Slice template — a[start:stop:step]

Three slots, all optional. Defaults flip with the sign of step.

a[ start : stop : step ]
   ↑       ↑      ↑
   first   one    direction & stride
   index   PAST   (default 1)
   to      the
   take    last
   (def 0) (def len(a))

a = [10, 20, 30, 40, 50]

# ── start/stop only (step omitted = 1) ──
a[1:4]      # → [20, 30, 40]   start at 1, STOP IS EXCLUSIVE (index 4 not included)
a[:3]       # → [10, 20, 30]   start defaults to 0
a[2:]       # → [30, 40, 50]   stop defaults to len(a)
a[:]        # → [10, 20, 30, 40, 50]   full shallow copy

# ── step = 1 (forward, default) ──
a[::1]      # → [10, 20, 30, 40, 50]   same as a[:]
a[1::1]     # → [20, 30, 40, 50]       from index 1 onward, every element

# ── step = 2 (forward, every other) ──
a[::2]      # → [10, 30, 50]           every 2nd element from the start
a[1::2]     # → [20, 40]               start at 1, every 2nd

# ── step = -1 (backward, full reverse) ──
a[::-1]     # → [50, 40, 30, 20, 10]   when step is negative, defaults flip:
            #                          start defaults to len(a)-1, stop to "before index 0"

# ── step = -2 (backward, every other) ──
a[::-2]     # → [50, 30, 10]           reverse, every 2nd element

# ── negative indices for start/stop ──
a[-2:]      # → [40, 50]               last two
a[:-2]      # → [10, 20, 30]           everything except the last two
a[-1:-4:-1] # → [50, 40, 30]           explicit reverse range
  • stop is exclusive — a[1:4] gives indices 1, 2, 3 (not 4). Same convention as range
  • Defaults flip with sign of step — positive step → start=0, stop=len(a); negative step → start=len(a)-1, stop="just past index 0"
  • step cannot be 0 — raises ValueError
  • Slicing never raises on out-of-range — a[100:200] returns [] instead of IndexError (unlike a[100]). Useful for safe substring/sublist work
  • A slice always returns a NEW list (shallow copy of selected elements); the original is never mutated
  • Strings and tuples slice with the same syntax — "hello"[::-1] → "olleh"

🎯 Range template — range(start, stop, step)

Same start/stop/step mental model as slicing — three call shapes (1, 2, or 3 args).

range( start , stop , step )
        ↑       ↑      ↑
        first   one    direction & stride
        value   PAST   (default 1, cannot be 0)
        (def 0) the
                last

# Three call shapes:
range(stop)              # 1 arg  — start=0, step=1
range(start, stop)       # 2 args — step=1
range(start, stop, step) # 3 args — full form

# ── Forward, single arg ──
list(range(5))             # → [0, 1, 2, 3, 4]    stop=5; 5 is NOT included

# ── Forward, two args ──
list(range(2, 7))          # → [2, 3, 4, 5, 6]    start=2, stop=7

# ── Forward, custom step ──
list(range(0, 10, 2))      # → [0, 2, 4, 6, 8]    step=2: every other
list(range(1, 10, 3))      # → [1, 4, 7]          step=3: every third

# ── Backward (negative step) ──
list(range(5, 0, -1))      # → [5, 4, 3, 2, 1]    stop=0 EXCLUDED — 0 not in result
list(range(5, -1, -1))     # → [5, 4, 3, 2, 1, 0] stop=-1 to INCLUDE 0

# ── Reverse-indices idiom (from practise/array.py exercise 8) ──
arr = [10, 20, 30]
for i in range(len(arr) - 1, -1, -1):
    print(i, arr[i])
# 2 30
# 1 20
# 0 10

# Annotated walkthrough of the reverse-indices call:
range( len(arr) - 1 , -1 , -1 )
         ↑            ↑     ↑
         last index   stop  step = -1
         (= 2 here)   one   (count
                      past  downward)
                      0
  • stop is exclusive — same convention as slicing. range(5) gives 0..4, never 5
  • step cannot be 0 — raises ValueError
  • Sign of step controls direction — positive counts up, negative counts down. If the direction can't reach stop (e.g. range(5, 0) with default step +1), the result is empty
  • range is lazy — returns a range object, not a list. Wrap in list(...) to materialize, or just iterate with for
  • Companion to slicing — same start, stop, step semantics, same exclusive-stop rule. Learning one teaches the other
  • ⚠️ Common bug: range(len(arr)-1, 0, -1) stops at 1 and misses index 0 (because stop is exclusive and 0 == stop). Use -1 as stop to include index 0

🔀 Sorted template — sorted(iterable, key=None, reverse=False)

Three slots: what to sort, how to compare each item, and direction. Returns a NEW list — original is untouched.

sorted( iterable , key=None , reverse=False )
          ↑          ↑           ↑
          what       how to      direction
          to sort    rank each   (False = ascending,
          (list,     item        True = descending)
          tuple,     (function
          set,       called per
          dict       element)
          keys, ...)

# ── No options: ascending, compare items directly ──
sorted([3, 1, 4, 1, 5])              # → [1, 1, 3, 4, 5]
sorted("banana")                     # → ['a', 'a', 'a', 'b', 'n', 'n']  (any iterable!)

# ── reverse=True: descending ──
sorted([3, 1, 4], reverse=True)        # → [4, 3, 1]

# ── key=func: rank by a transformed value ──
sorted(["pear", "fig", "apple"], key=len)   # → ['fig', 'pear', 'apple']  (by length)
sorted([3, -1, 4, -5], key=abs)        # → [-1, 3, 4, -5]            (by absolute value)

# ── key=lambda: inline custom rank ──
sorted([3, 1, 5], key=lambda x: -x)   # → [5, 3, 1]   (descending via negation — same as reverse=True for numbers)

# ── Multi-key (tuple): primary key, then tie-breaker ──
people = [(25, "Bob"), (30, "Alice"), (25, "Eve")]
sorted(people, key=lambda x: (-x[0], x[1]))
# → [(30, 'Alice'), (25, 'Bob'), (25, 'Eve')]
#   age DESC (negate), name ASC on ties

# ── In-place vs new list ──
arr = [3, 1, 2]
new = sorted(arr)              # → [1, 2, 3]   arr unchanged
arr.sort()                       # mutates arr → [1, 2, 3]; returns None — DON'T chain
  • sorted returns a NEW list; list.sort() mutates in place and returns None (so x = arr.sort() is the #1 bug)
  • key is a function, not a value — Python calls it once per element and uses the return value to rank. The original elements are kept in the result
  • reverse=True flips the whole order. For mixed directions (one key asc, another desc), put the negation inside the key tuple instead
  • Sort is stable — equal keys preserve original order. Lets you do multi-pass sorts: sort by secondary key first, then primary
  • Same key= / reverse= arguments work on min(), max(), heapq.nlargest(), heapq.nsmallest()

🌟 Star unpacking — *iterable (positional)

* spreads an iterable into positional slots — at a function call, in a literal, or on the left-hand side of an assignment. Mirror image of *args in a definition.

# Two contexts — mirror images:
# At a CALL site (spread out):
f(*iterable)               # positional unpack: spreads items as args

# In a DEFINITION (collect up):
def f(*args): ...        # collects extra positionals into a tuple

# ── 1) Function-call spread ──
def add(a, b, c): return a + b + c
nums = [1, 2, 3]
add(*nums)                 # → 6     same as add(1, 2, 3)
add(nums)                  # → TypeError — passes the LIST as one arg

# ── 2) Matrix transpose: zip(*matrix) ──
matrix = [[1, 2, 3], [4, 5, 6]]
list(zip(*matrix))         # → [(1, 4), (2, 5), (3, 6)]   columns become rows
# * unpacks the rows into zip's positional slots:
#   zip(*matrix) ≡ zip([1,2,3], [4,5,6])

# ── 3) List/tuple literal merge (PEP 448) ──
a = [1, 2]
b = [3, 4]
[*a, *b]                   # → [1, 2, 3, 4]              merge sequences
[0, *a, 99, *b]            # → [0, 1, 2, 99, 3, 4]       interleave with literals
(*a, *b)                   # → (1, 2, 3, 4)              same trick for tuples

# ── 4) LHS unpacking — split off head/tail ──
first, *rest = [1, 2, 3, 4]
# first = 1, rest = [2, 3, 4]

*head, last = [1, 2, 3, 4]
# head = [1, 2, 3], last = 4

a, *mid, z = [1, 2, 3, 4, 5]
# a = 1, mid = [2, 3, 4], z = 5

# ── 5) Definition-time collection ──
def total(*nums):           # nums is a TUPLE of all positional args
    return sum(nums)
total(1, 2, 3, 4)            # → 10
  • Two roles — call-site spread (open up) vs definition-time collect (gather up). Same syntax, opposite directions
  • list(zip(*matrix)) is the standard idiom for transposing a matrix — no NumPy needed. See "🧮 Matrix & Index Tricks" tips-box below for the full pattern
  • [*a, *b] and (*a, *b) are usually clearer/faster than a + b for sequence merging — and they work across mixed iterable types
  • Companion: ** for keyword/dict unpacking — see dict.html → Syntax Templates

📄 Shallow copy — example

Make an independent list so changes to the copy don't touch the original.

a = [1, 2, 3]
b = a.copy()        # or a[:]  or list(a)  — all equivalent
b.append(99)

# a → [1, 2, 3]        (unchanged)
# b → [1, 2, 3, 99]
  • A shallow copy is enough for flat lists of immutables (numbers, strings, tuples)
  • It breaks when the list contains other lists/dicts/objects — see deep copy below

🌳 Deep copy — example (and why shallow breaks)

When the list contains other lists, a shallow copy duplicates the outer list but still shares the inner ones.

# ── Shallow copy of a nested list — TRAP ──
a = [[1, 2], [3, 4]]
b = a.copy()        # shallow — inner lists are SHARED
b[0].append(99)

# a → [[1, 2, 99], [3, 4]]   ← a was mutated too!
# b → [[1, 2, 99], [3, 4]]

# ── Deep copy — independent inner lists ──
import copy
a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)
b[0].append(99)

# a → [[1, 2], [3, 4]]       (unchanged)
# b → [[1, 2, 99], [3, 4]]
  • Reach for copy.deepcopy when the list contains lists, dicts, or objects you intend to mutate
  • For flat lists of numbers/strings, deepcopy is overkill — .copy() is fine

➕ append vs extend — example

append(b) adds b as one element. extend(b) adds each item of b.

a = [1, 2]
b = [3, 4]

x = a.copy(); x.append(b)    # → [1, 2, [3, 4]]   b added as ONE element (a list inside a list)
y = a.copy(); y.extend(b)    # → [1, 2, 3, 4]     b's items added one by one
z = a.copy(); z += b         # → [1, 2, 3, 4]     same as extend
  • append(x) always adds exactly one element — even if x is itself a list
  • extend(iterable) iterates and appends each item — works on lists, tuples, strings, generators
  • a += b on a list is identical to a.extend(b) (mutates in place)
  • c = a + b builds a NEW list — a and b stay unchanged
🐍 Python
nums = []                          # empty list
nums = [1, 2, 3]                  # literal
nums = [0] * 10                   # 10 zeros
nums = list(range(5))            # [0,1,2,3,4]
nums = [x**2 for x in range(5)]  # [0,1,4,9,16]

# ── 2D List (Matrix) ──
grid = [[0] * cols for _ in range(rows)]
# ⚠️ WRONG: [[0]*cols]*rows (shared refs!)
☕ Java
List<Integer> nums = new ArrayList<>();
List<Integer> nums = new ArrayList<>(Arrays.asList(1,2,3));
List<Integer> nums = List.of(1,2,3); // immutable!
int[] arr = new int[10];           // 10 zeros
// No built-in range → use IntStream

// ── 2D Array ──
int[][] grid = new int[rows][cols];
// Or with ArrayList:
// List<List<Integer>> grid = new ArrayList<>();

⚠️ 2D Grid Gotcha

  • [[0]*cols]*rows creates rows references to the SAME inner list — mutating one row mutates them all
  • Always use a comprehension: [[0]*cols for _ in range(rows)]

📋 Copy & extend nuances

# ── Append vs allocate-new ──
a.extend(b)                      # mutate in place (same as: a += b)
a += b                           # same as extend on a list
c = a + b                        # NEW list; a, b unchanged — use when you must preserve a

# ── Three shallow-copy idioms (all equivalent) ──
b = a[:]                         # slice copy — terse, very common
b = a.copy()                     # method form (3.3+) — most readable
b = list(a)                      # works for any iterable, not just lists

# ── Deep copy for nested structures ──
import copy
b = copy.deepcopy(a)             # required when a contains lists/dicts/objects

# ── Reverse: in-place vs new ──
a.reverse()                      # mutates a (returns None — don't chain!)
b = a[::-1]                      # NEW reversed list — leaves a alone
for x in reversed(a): ...        # iterator only, no copy — fastest if you only iterate

For why a = b isn't a copy at all (and bites you in interviews), see Common Gotchas → List Copy vs Reference.

🛠️ Built-ins commonly paired with lists

When you write list code in interviews, these are the built-ins you reach for most.

# ── Iterating ──
for v in arr: ...
for i, v in enumerate(arr): ...           # index + value
for v in reversed(arr): ...               # iterator, no copy
for a, b in zip(arr, arr[1:]): ...        # adjacent pairs

# ── Searching ──
target in arr                              # O(n) membership
arr.index(x)                              # first index of x (raises if missing)
next((i for i, v in enumerate(arr) if v > k), -1)  # find-first index, safe default

# ── Sorting ──
sorted(arr, key=lambda x: -x)              # custom key, NEW list
sorted(arr, key=lambda x: (-x[0], x[1]))   # multi-key
arr.sort()                                # in place, returns None — don't chain!

# ── Aggregating ──
sum(arr), min(arr), max(arr)
sum(arr, start=10)                        # offset
any(x > 0 for x in arr)
all(x > 0 for x in arr)

# ── Transforming ──
[x*x for x in arr]                        # comprehension — list (materialized)
(x*x for x in arr)                        # generator — lazy, one value at a time
list(map(str, arr)), list(filter(None, arr))

🔍 Decoding next((i for i, v in enumerate(arr) if v > k), -1)

"Index of the first element greater than k, or -1 if none." Three layers stacked:

  1. enumerate(arr) yields (index, value) pairs as you iterate.
  2. (i for i, v in enumerate(arr) if v > k) is a generator expression — it yields just the i (index) whenever v > k. Lazy: nothing runs until something asks for the next value.
  3. next(generator, default) pulls the first item the generator produces. If the generator is empty (no match), it returns the default instead of raising StopIteration.
arr = [3, 1, 7, 2, 9]; k = 5
next((i for i, v in enumerate(arr) if v > k), -1)   # → 2  (arr[2]=7 is first > 5)

arr = [3, 1, 2]; k = 5
next((i for i, v in enumerate(arr) if v > k), -1)   # → -1 (no match)

# Equivalent loop:
def find_first(arr, k):
    for i, v in enumerate(arr):
        if v > k: return i
    return -1
  • Short-circuits — stops at the first match, doesn't scan the rest of arr
  • No StopIteration leak — the -1 default makes it safe on empty matches
  • ⚠️ The two paren layers are mandatory: inner (...) is the generator expression, outer (...) is next()'s argument list. Forgetting the default (next(gen) alone) is the #1 bug

📝 First, what's a lambda?

A lambda is a tiny anonymous function — one expression, no def, no name. It's used wherever a callable is expected for a single line. 

# Syntax:  lambda <params>: <expression>
#          ↑ keyword     ↑ what the function returns (no `return` keyword)

lambda x: x * 2           # a function that returns 2x
lambda x, y: x + y       # two args
lambda: 42               # no args, returns 42

# Equivalent named version:
def double(x):
    return x * 2

# Why use it: pass a function inline without ceremony.
sorted(arr, key=lambda x: -x)            # inline — clean

def neg(x): return -x                      # named — works but verbose
sorted(arr, key=neg)
  • Body is a single expression — no statements, no multi-line logic. If you need that, write a regular def
  • Result is the value of that expression — there's no return
  • Common partners: sorted(... key=), min/max(... key=), map, filter, functools.reduce, heapq.nlargest(... key=)
  • Don't assign a lambda to a name (double = lambda x: x*2) — just use def. Style guides flag this

🔀 Decoding sorted(arr, key=lambda x: -x) and multi-key sorts

key= takes a function. sorted calls it on every element and uses the returned value as the sort key — but returns the original elements in that order. 

arr = [3, 1, 4, 1, 5]

sorted(arr)                       # [1, 1, 3, 4, 5]   ascending (default)
sorted(arr, key=lambda x: -x)     # [5, 4, 3, 1, 1]   descending — negation flips the order

# How the negation trick works:
# • transformed keys: [-3, -1, -4, -1, -5]
# • sorted asc by key: [-5, -4, -3, -1, -1]
# • original elements in that order: [5, 4, 3, 1, 1]   ← the result

# For numbers, this equals sorted(arr, reverse=True). The trick matters in MULTI-KEY sorts.

Multi-key: when the key returns a tuple, Python compares tuples lexicographically — first item, then second on ties, etc. 

people = [(25, "Bob"), (30, "Alice"), (25, "Eve"), (30, "Tom")]

sorted(people, key=lambda x: (-x[0], x[1]))
# → [(30, 'Alice'), (30, 'Tom'), (25, 'Bob'), (25, 'Eve')]
#
# • -x[0]:  30 → -30, 25 → -25;  -30 < -25, so 30s come first   (age DESC)
# • Within ties, x[1] sorts ascending alphabetically               (name ASC)

🎯 Why the negation trick (and when it breaks)

reverse=True flips the whole ordering. For mixed directions — one key asc, another desc — you can't use it. Negating numerics inside the tuple is the standard workaround:

GoalKey
age asc, name asclambda x: (x[0], x[1])
age desc, name asclambda x: (-x[0], x[1])
age asc, name desctwo-pass sort (see below) — strings have no -
age desc, name desclambda x: (x[0], x[1]) with reverse=True

⚠️ Negation only works on numerics. Strings have no - operator. For string columns, sort twice in reverse priority order — Python's sort is stable, so the second pass preserves the first key's order within ties: 

# "age asc, name desc":
data = sorted(people, key=lambda x: x[1], reverse=True)   # name desc first
data = sorted(data,   key=lambda x: x[0])                  # then age asc — wins on ties

The key= argument also works on min(), max(), heapq.nlargest(), and heapq.nsmallest() — same trick.

Operations

🐍 Python
nums = [3, 1, 4, 1, 5]

# ── Add ──
nums.append(9)              # [3,1,4,1,5,9]
nums.insert(0, 7)           # [7,3,1,4,1,5,9]
nums.extend([2, 6])         # append multiple

# ── Access ──
nums[0]                     # first → 7
nums[-1]                    # last → 6
nums[1:4]                   # slice → [3,1,4]

# ── Remove ──
nums.pop()                  # remove last → 6
nums.pop(0)                 # remove first → 7
nums.remove(4)              # remove first occurrence of 4

# ── Search ──
1 in nums                  # True (O(n))
nums.index(1)               # first index of 1
nums.count(1)               # how many 1s

# ── Sort ──
nums.sort()                 # in-place ascending
nums.sort(reverse=True)     # in-place descending
sorted(nums)                # new sorted list
sorted(nums, key=lambda x: -x)  # custom sort

# ── Reverse ──
nums.reverse()              # in-place
nums[::-1]                  # new reversed list

# ── Size ──
len(nums)                   # length

# ── Comprehension (Python superpower!) ──
squares = [x**2 for x in range(10)]
evens = [x for x in nums if x % 2 == 0]
flat = [x for row in grid for x in row]
☕ Java
List<Integer> nums = new ArrayList<>(
    Arrays.asList(3, 1, 4, 1, 5));

// ── Add ──
nums.add(9);                // [3,1,4,1,5,9]
nums.add(0, 7);             // [7,3,1,4,1,5,9]
nums.addAll(Arrays.asList(2, 6)); // append multiple

// ── Access ──
nums.get(0);                // first → 7
nums.get(nums.size()-1);    // last → 6
nums.subList(1, 4);         // slice → [3,1,4]

// ── Remove ──
nums.remove(nums.size()-1); // remove last
nums.remove(0);             // remove at index 0
nums.remove(Integer.valueOf(4)); // remove value 4

// ── Search ──
nums.contains(1);           // true (O(n))
nums.indexOf(1);            // first index of 1
Collections.frequency(nums, 1); // count of 1

// ── Sort ──
Collections.sort(nums);     // in-place ascending
Collections.sort(nums, Collections.reverseOrder());
nums.stream().sorted().collect(Collectors.toList());
nums.sort((a,b) -> b - a);  // custom sort

// ── Reverse ──
Collections.reverse(nums);  // in-place
// No one-liner for new reversed list

// ── Size ──
nums.size();                // length

// ── Stream API (closest to comprehension) ──
int[] squares = IntStream.range(0,10)
    .map(x -> x*x).toArray();
// nums.stream().filter(x -> x%2==0).toList();

⏪ Reverse Iteration Idioms — pick the right form

Several Python forms iterate "backward" but they're not interchangeable. Choose by what you need: indices vs values, full vs sliced, custom step. 

# 1) Walk indices last → first  (when you need i, not arr[i] alone)
for i in range(len(arr) - 1, -1, -1):
    ...   # i = 5,4,3,2,1,0  for arr of length 6

# 2) Walk values last → first  (when index doesn't matter)
for v in reversed(arr):
    ...

# 3) Walk a slice in reverse  (skip arr[0], or any sub-range)
for v in reversed(arr[1:]):
    ...   # yields arr[-1], arr[-2], ..., arr[1]

# 4) Same as (1), more explicit about "reverse of forward range"
for i in reversed(range(len(arr))):
    ...

# 5) New reversed list (allocates a copy — usually overkill)
for v in arr[::-1]:
    ...

# 6) Backward with custom step (only range() supports this)
for i in range(len(arr) - 1, -1, -2):
    ...   # every other index, last → first

📐 Why range(len(arr)-1, -1, -1) uses -1 as stop

  • start = len(arr) - 1 → last valid index
  • stop = -1 → "one past 0 going backward" — because stop is exclusive
  • step = -1 → count downward
  • ⚠️ range(len(arr)-1, 0, -1) is the common bug — it stops at 1 and misses index 0

🎯 When to use which

  • Need the index (e.g. writing back to arr[i], two-pointer math) → form (1) or (4)
  • Need the value only → form (2); it's an iterator (no copy) and reads cleanest
  • Need to skip a prefix/suffix in reverse → form (3): reversed(arr[1:]), reversed(arr[:-1]), etc. (slice allocates a small copy)
  • Need a non-default step (every other, every third) → form (6); reversed() can't express this
  • Avoid form (5) arr[::-1] in loops — it allocates a full reversed copy

🧮 Matrix & Index Tricks — zip(*m), divmod, ~i

Three idioms that compress matrix rotation, grid problems, and palindrome two-pointer code from many lines to one. Heavily used in Rotate Image, Transpose Matrix, Word Search, Spiral Matrix, Valid Palindrome. 

# ── 1) zip(*matrix) — transpose / iterate columns ──
matrix = [[1,2,3], [4,5,6], [7,8,9]]
list(zip(*matrix))             # [(1,4,7),(2,5,8),(3,6,9)]
# Rotate 90° clockwise (LC 48): transpose then reverse each row
rotated = [list(reversed(row)) for row in zip(*matrix)]

# Iterate column-by-column — no nested index gymnastics
for col in zip(*matrix):
    print(sum(col))             # sum each column

# ── 2) divmod(i, cols) — flat index ↔ (row, col) ──
# Lets you use a single int set/dict for visited (faster hashing
# than tuple keys) on rows × cols grids.
for i in range(rows * cols):
    r, c = divmod(i, cols)         # i=7, cols=3 → r=2, c=1
    # grid[r][c] ...

# Inverse: flat = r * cols + c

# ── 3) arr[~i] — i-th from the end (~i == -i-1) ──
# arr[~0] == arr[-1], arr[~1] == arr[-2], ...
# Palindrome check without len() arithmetic:
for i in range(len(s) // 2):
    if s[i] != s[~i]:                # ← i and ~i meet in the middle
        return False

# Rotate-image inner swap (LC 48): cleaner than matrix[n-1-j][i]
matrix[i][j], matrix[~j][i], matrix[~i][~j], matrix[j][~i] = \
    matrix[~j][i], matrix[~i][~j], matrix[j][~i], matrix[i][j]

💡 Why these matter

  • zip(*matrix) works because * unpacks the rows as positional args to zip — Python's transpose, no NumPy needed
  • divmod + flat indexing is the standard speedup for BFS/DFS on grids when you want a set[int] visited instead of set[tuple]
  • ~i reads as "complement of i" — for any non-negative i, ~i equals -i-1, which Python interprets as "i-th from the end". Replaces every n - 1 - i in your code
