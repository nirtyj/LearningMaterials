1. Create

⚠️ {} is an empty dict, not a set

Use set() for an empty set. {1} is a set; {} is a dict; {"a": 1} is a dict. Python's parser commits when it sees a colon.

2. Get / Lookup

Read an element you already know how to address. Raise vs safe-default is the most common pitfall.

CollectionStrict (raises if missing)Safe (returns default)Peek / smallest
lista[i] → IndexErrora[i] if i < len(a) else defaulta[0], a[-1]
dictd[k] → KeyErrord.get(k, default)—
set— (use in)— (use in)—
strs[i] → IndexErrors[i:i+1] (slice — never raises)s[0], s[-1]
dequedq[i] → IndexError—dq[0], dq[-1]
Counter—c[k] → 0 if missing ✨c.most_common(1)[0]
heap——h[0] (smallest, no pop)

🛡️ d[k] vs d.get(k)

Use d[k] when missing means a bug (let it crash). Use d.get(k, default) when missing is normal (counting, optional config). Same shape as list.index(x) vs "abc".find(x).

✨ Counter[k] returns 0 on miss — that's the whole point

Accessing a missing key on a Counter does NOT insert it and does NOT raise. c["x"] returns 0. "x" in c is still False afterward.

4. Add / Insert

Put a new item in. "Add one" appends/sets a single element; "Bulk" takes an iterable.

CollectionAdd oneAt a positionBulk
lista.append(x)a.insert(i, x)a.extend(iter) · a += iter
dictd[k] = v · d.setdefault(k, v)— (no order to position into before 3.7)d.update(other) · d |= other (3.9+)
sets.add(x)— (unordered)s.update(*iters) · s |= other
dequedq.append(x) · dq.appendleft(x)— (use rotate)dq.extend(iter) · dq.extendleft(iter)
Counterc[k] += 1—c.update(iter) · c.subtract(iter)
heapheapq.heappush(h, x)—build then heapq.heapify(h)

📌 append vs extend (list / deque)

a.append(b) adds b as one element (the list becomes nested). a.extend(b) iterates b and adds each item.

a = [1, 2]; b = [3, 4]
a.copy() + [b]   # [1, 2, [3, 4]]   append shape
a + b            # [1, 2, 3, 4]     extend shape

🪤 dq.extendleft(iter) reverses the input

Each item is appendleft'd in turn — so the last item ends up leftmost. deque([1]); dq.extendleft([2,3]) → deque([3, 2, 1]).

🌱 setdefault mutates

d.setdefault(k, default) returns the existing value if k is present, otherwise inserts default and returns it. Different from d.get(k, default), which never inserts.

6. Remove / Delete

Delete an element. Raise vs silent on a missing target is the central confusion.

CollectionBy value — raisesBy value — silentBy index / keyClear all
lista.remove(x) → ValueError— (use a guard: if x in a: a.remove(x))del a[i] · a.pop(i)a.clear()
dict——del d[k] (raise) · d.pop(k, default) (safe)d.clear()
sets.remove(x) → KeyErrors.discard(x) ← silent—s.clear()
dequedq.remove(x) → ValueError—del dq[i]dq.clear()
str—s.replace(old, "") (immutable → new string)——

⚠️ set.discard(x) vs set.remove(x)

discard = "delete if present" (silent on miss); remove = "must be there, crash if not". Same arg shape, different miss behavior.

⚠️ list.remove(x) takes a value; list.pop(i) takes an index

Easy to invert: a.remove(0) deletes the first value equal to 0, NOT the first element. To delete by index, use a.pop(0) or del a[0].

🛡️ d.pop(k, default) never raises

Pass a sentinel default (commonly None) when the key may not exist. d.pop(k) alone raises KeyError on miss.

7. Pop — remove and return

Combined "delete + give it to me". Each collection picks a different element by default.

CollectionDefault pop returns…Other ends / variants
lista.pop() → last (O(1))a.pop(0) → first (O(n)!) · a.pop(i)
dictd.pop(k) → value at k (raises)d.pop(k, default) · d.popitem() (LIFO, raise if empty)
sets.pop() → arbitrary element (raise if empty)— (use discard for targeted)
dequedq.pop() → rightmost (O(1))dq.popleft() → leftmost (O(1))
heapheapq.heappop(h) → smallest (O(log n))heapq.heappushpop(h, x) · heapq.heapreplace(h, x)

🚫 list.pop(0) is O(n) — use deque.popleft() instead

Removing from the front of a list shifts every element. For a FIFO queue (BFS), use collections.deque — both append and popleft are O(1).

🔍 str.find vs str.index

Same args, different miss behavior — exactly like dict.get vs d[k] and set.discard vs set.remove. It's the same pattern everywhere.

11. Sort / Order

sorted() always returns a NEW list; .sort() mutates in place and returns None.

CollectionNew sortedIn-placeBy key / descending
listsorted(a)a.sort()sorted(a, key=len, reverse=True)
dictsorted(d) (sorts keys)— (insertion-ordered in 3.7+)sorted(d.items(), key=lambda x: x[1])
set / frozensetsorted(s) → list— (unordered)sorted(s, reverse=True)

heap—heapq.heapify(list) (heap order, NOT sorted)heapq.nlargest(n, iter, key=...) · heapq.nsmallest(...)

⚠️ x = a.sort() is the #1 bug

list.sort() returns None and mutates a in place. Same for heapq.heapify(), list.reverse(), random.shuffle(). If you want a new sorted list, use sorted(a).

12. Copy

Make an independent collection. Shallow duplicates the outer container; deep recursively duplicates inner objects too.

CollectionShallowDeep
lista.copy() · a[:] · list(a)copy.deepcopy(a)
dictd.copy() · dict(d) · {**d}copy.deepcopy(d)
sets.copy() · set(s)copy.deepcopy(s)
dequedq.copy() · deque(dq)copy.deepcopy(dq)
tuple / str / frozensetnot needed — immutable, sharing is safe—

🌳 When you NEED deep copy

Shallow is fine for flat containers of immutables (numbers, strings, tuples). The moment any element is itself a mutable container (list of lists, dict of lists, etc.) and you'll mutate it, you need copy.deepcopy. See list.html for the worked example.

13. Combine / Merge

Build a NEW collection from two existing ones. Operators usually return new; the = variants mutate in place.

CollectionNewIn place
list / tuple / stra + ba += b (list/deque only; for tuple/str this rebinds)
dict{**d1, **d2} · d1 | d2 (3.9+)d1.update(d2) · d1 |= d2
set — unions1 | s2s1 |= s2 · s1.update(s2)
set — intersections1 & s2s1 &= s2
set — differences1 - s2s1 -= s2
set — symmetric diffs1 ^ s2s1 ^= s2
Counterc1 + c2 · c1 - c2 · c1 & c2 (min) · c1 | c2 (max)c1.update(c2) (add) · c1.subtract(c2) (allow negatives)
heaplist(heapq.merge(h1, h2)) (inputs must be sorted)—

🔀 Counter & ≠ set &

Sets: {1,2,3} & {2,3,4} → {2, 3} (intersection). Counters: Counter("aab") & Counter("abb") → Counter({'a':1,'b':1}) (elementwise MIN). Same operator, different semantics — multiset vs boolean.

⚠️ dict.update(other) overwrites — Counter.update(other) ADDS

Same method name, different semantics. dict.update replaces colliding values; Counter.update sums them.

Which collection should I use?

🧭 When in doubt — by what you need

  • Ordering + indexed access → list
  • O(1) key lookup by name → dict
  • Uniqueness + fast membership → set
  • O(1) push/pop at BOTH ends → collections.deque
  • Counts / frequencies → collections.Counter
  • Auto-init missing keys (grouping/counting) → collections.defaultdict
  • Smallest-first priority queue → heapq on a list
  • Hashable / immutable sequence → tuple (dict keys, set elements, return-multiple-values)
  • Hashable / immutable set → frozenset
