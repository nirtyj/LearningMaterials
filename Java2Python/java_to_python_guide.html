💡 Interview Insight: Mutability

  • Python: Lists are mutable, tuples/strings are immutable
  • Java: Arrays are mutable, Strings are immutable
  • In Python, a = b for lists creates a reference, not a copy!
  • Use a = b.copy() or a = b[:] for shallow copy

⏪ Reverse Iteration Idioms — pick the right form

Several Python forms iterate "backward" but they're not interchangeable. Choose by what you need: indices vs values, full vs sliced, custom step. 

# 1) Walk indices last → first  (when you need i, not arr[i] alone)
for i in range(len(arr) - 1, -1, -1):
    ...   # i = 5,4,3,2,1,0  for arr of length 6

# 2) Walk values last → first  (when index doesn't matter)
for v in reversed(arr):
    ...

# 3) Walk a slice in reverse  (skip arr[0], or any sub-range)
for v in reversed(arr[1:]):
    ...   # yields arr[-1], arr[-2], ..., arr[1]

# 4) Same as (1), more explicit about "reverse of forward range"
for i in reversed(range(len(arr))):
    ...

# 5) New reversed list (allocates a copy — usually overkill)
for v in arr[::-1]:
    ...

# 6) Backward with custom step (only range() supports this)
for i in range(len(arr) - 1, -1, -2):
    ...   # every other index, last → first

📐 Why range(len(arr)-1, -1, -1) uses -1 as stop

  • start = len(arr) - 1 → last valid index
  • stop = -1 → "one past 0 going backward" — because stop is exclusive
  • step = -1 → count downward
  • ⚠️ range(len(arr)-1, 0, -1) is the common bug — it stops at 1 and misses index 0

🎯 When to use which

  • Need the index (e.g. writing back to arr[i], two-pointer math) → form (1) or (4)
  • Need the value only → form (2); it's an iterator (no copy) and reads cleanest
  • Need to skip a prefix/suffix in reverse → form (3): reversed(arr[1:]), reversed(arr[:-1]), etc. (slice allocates a small copy)
  • Need a non-default step (every other, every third) → form (6); reversed() can't express this
  • Avoid form (5) arr[::-1] in loops — it allocates a full reversed copy

🧮 Matrix & Index Tricks — zip(*m), divmod, ~i

Three idioms that compress matrix rotation, grid problems, and palindrome two-pointer code from many lines to one. Heavily used in Rotate Image, Transpose Matrix, Word Search, Spiral Matrix, Valid Palindrome. 

# ── 1) zip(*matrix) — transpose / iterate columns ──
matrix = [[1,2,3], [4,5,6], [7,8,9]]
list(zip(*matrix))             # [(1,4,7),(2,5,8),(3,6,9)]
# Rotate 90° clockwise (LC 48): transpose then reverse each row
rotated = [list(reversed(row)) for row in zip(*matrix)]

# Iterate column-by-column — no nested index gymnastics
for col in zip(*matrix):
    print(sum(col))             # sum each column

# ── 2) divmod(i, cols) — flat index ↔ (row, col) ──
# Lets you use a single int set/dict for visited (faster hashing
# than tuple keys) on rows × cols grids.
for i in range(rows * cols):
    r, c = divmod(i, cols)         # i=7, cols=3 → r=2, c=1
    # grid[r][c] ...

# Inverse: flat = r * cols + c

# ── 3) arr[~i] — i-th from the end (~i == -i-1) ──
# arr[~0] == arr[-1], arr[~1] == arr[-2], ...
# Palindrome check without len() arithmetic:
for i in range(len(s) // 2):
    if s[i] != s[~i]:                # ← i and ~i meet in the middle
        return False

# Rotate-image inner swap (LC 48): cleaner than matrix[n-1-j][i]
matrix[i][j], matrix[~j][i], matrix[~i][~j], matrix[j][~i] = \
    matrix[~j][i], matrix[~i][~j], matrix[j][~i], matrix[i][j]

💡 Why these matter

  • zip(*matrix) works because * unpacks the rows as positional args to zip — Python's transpose, no NumPy needed
  • divmod + flat indexing is the standard speedup for BFS/DFS on grids when you want a set[int] visited instead of set[tuple]
  • ~i reads as "complement of i" — for any non-negative i, ~i equals -i-1, which Python interprets as "i-th from the end". Replaces every n - 1 - i in your code

📚 Stack Operations

🐍 Python
stack = []

# ── Push / Pop / Peek ──
stack.append(1)              # push → [1]
stack.append(2)              # push → [1,2]
stack.append(3)              # push → [1,2,3]
stack[-1]                    # peek → 3 (no remove)
stack.pop()                  # pop → 3, stack=[1,2]

# ── Check ──
len(stack) == 0              # is empty?
not stack                    # Pythonic empty check

# ── Common Pattern: Monotonic Stack ──
stack = []
for i, num in enumerate(nums):
    while stack and nums[stack[-1]] < num:
        stack.pop()
    stack.append(i)
☕ Java
Deque<Integer> stack = new ArrayDeque<>();

// ── Push / Pop / Peek ──
stack.push(1);               // push → [1]
stack.push(2);               // push → [2,1]
stack.push(3);               // push → [3,2,1]
stack.peek();                // peek → 3 (no remove)
stack.pop();                 // pop → 3

// ── Check ──
stack.isEmpty();             // is empty?
stack.size();                // size

// ── Common Pattern: Monotonic Stack ──
Deque<Integer> stack = new ArrayDeque<>();
for (int i = 0; i < nums.length; i++) {
    while (!stack.isEmpty() &&
           nums[stack.peek()] < nums[i])
        stack.pop();
    stack.push(i);
}

⛰️ Heap / PriorityQueue Operations

🐍 Python
import heapq

# ── Min Heap ──
heap = []
heapq.heappush(heap, 5)      # [5]
heapq.heappush(heap, 1)      # [1,5]
heapq.heappush(heap, 3)      # [1,5,3]
heap[0]                      # peek min → 1
heapq.heappop(heap)          # pop min → 1

# ── Max Heap (negate values!) ──
heapq.heappush(heap, -val)
-heapq.heappop(heap)

# ── Heapify existing list ──
nums = [5, 3, 8, 1]
heapq.heapify(nums)          # O(n), in-place

# ── Push + Pop combined ──
heapq.heappushpop(heap, val) # push then pop min
heapq.heapreplace(heap, val) # pop min then push

# ── Top K Pattern ──
heapq.nlargest(3, nums)      # 3 largest
heapq.nsmallest(3, nums)     # 3 smallest

# ── With tuples (priority, data) ──

# (a) Sort the word — O(k log k) per word, simplest:

# into sorted()/heap operations.

# ── 5) Sliding-window anagram check (LC 438) ──
# Compare two Counters in O(26) — Counter equality is set-based.
need = Counter(p)
window = Counter(s[:len(p)])
if window == need: ans.append(0)
for i in range(len(p), len(s)):
    window[s[i]] += 1
    window[s[i - len(p)]] -= 1
    if window[s[i - len(p)]] == 0:
        del window[s[i - len(p)]]   # ← keep == comparison clean
    if window == need: ans.append(i - len(p) + 1)

# ── 6) dict.fromkeys(arr) — order-preserving dedup ──
# Java's LinkedHashSet in three tokens (3.7+ dicts are ordered).
list(dict.fromkeys([3,1,3,2,1]))  # [3, 1, 2]

⚠️ Gotchas

  • list is unhashable → can't use list as dict key. Always wrap a frequency vector in tuple(...) before using it as a key
  • When comparing Counters with ==, delete zero-count keys (or use +window to drop them) — otherwise Counter({'a':1, 'b':0}) != Counter({'a':1})
  • ord('a') == 97, ord('A') == 65 — for mixed case, normalize with .lower() first or use a 52-slot vector
  • Sorted-tuple key is O(k log k) per word but reads cleanest; count-vector key is O(k) — pick by input size, not aesthetics

def build_tree(vals):
    if not vals: return None
    root = TreeNode(vals[0])

            q.append(node.left)
        i += 1
        if i < len(vals) and vals[i] is not None:
            node.right = TreeNode(vals[i])
            q.append(node.right)
        i += 1
    return root

def threeSum(nums):
    nums.sort()
    res = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]:
            continue                     # ← skip duplicates
        l, r = i + 1, len(nums) - 1
        while l < r:
            s = nums[i] + nums[l] + nums[r]
            if s < 0: l += 1
            elif s > 0: r -= 1
            else:
                res.append([nums[i], nums[l], nums[r]])
                while l < r and nums[l] == nums[l+1]: l += 1
                l += 1; r -= 1
    return res

# Trick: Counter for need vs window tracking
def minWindow(s, t):
    need = Counter(t)
    missing = len(t)
    l = start = 0; end = float('inf')
    for r, c in enumerate(s):
        if need[c] > 0: missing -= 1
        need[c] -= 1
        while missing == 0:              # all chars found
            if r - l < end - start:
                start, end = l, r
            need[s[l]] += 1
            if need[s[l]] > 0: missing += 1
            l += 1
    return s[start:end+1] if end < float('inf') else ""

🎯 When to reach for it

  • Need insert + delete + min/max all in O(log n) → SortedList (heap can't delete arbitrary)
  • Need order statistics (the k-th smallest) → SortedList (heap can't index)
  • Need range queries on a sorted multiset → SortedList.irange()
  • Plain bisect + list is fine when you only append/replace at known indices (LIS) — the O(n) insert middle is what kills you, and that's exactly what SortedList avoids
  • Interview note: most interviewers accept SortedList at FAANG. If forbidden, the fallback is two heaps (median) or a manually-balanced BST / segment tree

# ── Template: Tree level-order BFS ──
from collections import deque
def levelOrder(root):
    if not root: return []
    res, q = [], deque([root])
    while q:
        level = []
        for _ in range(len(q)):         # ← process one level
            node = q.popleft()
            level.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        res.append(level)
    return res

# ── Template: Graph/Grid BFS ──
def bfs(grid, start):
    q = deque([start])
    visited = {start}                    # ← set for O(1) lookup
    steps = 0
    while q:
        for _ in range(len(q)):
            r, c = q.popleft()           # ← tuple unpacking
            for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:  # ← 4 directions
                nr, nc = r + dr, c + dc
                if 0 <= nr < rows and 0 <= nc < cols \
                   and (nr, nc) not in visited:
                    visited.add((nr, nc)) # ← tuple as set element
                    q.append((nr, nc))
        steps += 1
    return steps

# • 0 <= nr < rows      — chained comparison

# ── Subsets (LC 78) ──
def subsets(nums):
    res = []
    def backtrack(start, path):
        res.append(path[:])              # ← copy! (path[:] or list(path))
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)
            path.pop()                   # ← undo choice (backtrack)
    backtrack(0, [])
    return res

# ── Permutations (LC 46) ──
def permute(nums):
    res = []
    def backtrack(path):
        if len(path) == len(nums):
            res.append(path[:])
            return
        for n in nums:
            if n not in path:            # ← O(n) but clean for interviews
                path.append(n)
                backtrack(path)
                path.pop()
    backtrack([])
    return res

# ── Combination Sum (LC 39) ──
def combinationSum(candidates, target):
    res = []
    def backtrack(start, path, total):
        if total == target:
            res.append(path[:])
            return
        for i in range(start, len(candidates)):
            if total + candidates[i] > target:
                break                    # ← prune (sort first!)
            path.append(candidates[i])
            backtrack(i, path, total + candidates[i])  # i not i+1 (reuse)
            path.pop()
    candidates.sort()                    # ← sort enables pruning
    backtrack(0, [], 0)
    return res

# ── Python tricks used ──
# • path[:]            — shallow copy of current path
# • path.append / pop  — mutate-then-undo pattern
# • sort + break       — prune search space
# • n not in path      — membership test in list

# ── Bottom-Up: Coin Change (LC 322) ──
def coinChange(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for a in range(1, amount + 1):
        for c in coins:
            if c <= a:
                dp[a] = min(dp[a], dp[a - c] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

# ── House Robber (LC 198) — space-optimized ──
def rob(nums):
    prev, curr = 0, 0
    for n in nums:
        prev, curr = curr, max(curr, prev + n)  # ← parallel assign!
    return curr

# ── LIS (LC 300) — patience sort with bisect ──
from bisect import bisect_left
def lengthOfLIS(nums):
    tails = []
    for n in nums:
        i = bisect_left(tails, n)
        if i == len(tails):
            tails.append(n)
        else:
            tails[i] = n                 # ← O(n log n)!
    return len(tails)

# ── 2D DP: LCS (LC 1143) ──
def longestCommonSubsequence(s1, s2):
    dp = [[0] * (len(s2)+1) for _ in range(len(s1)+1)]
    for i in range(1, len(s1)+1):
        for j in range(1, len(s2)+1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[-1][-1]                    # ← dp[-1][-1] = bottom-right

# • bisect_left          — O(n log n) LIS
# • dp[-1][-1]           — negative indexing for last element

# ── Topological Sort — BFS / Kahn's (LC 207) ──
def canFinish(numCourses, prerequisites):
    graph = defaultdict(list)
    indegree = [0] * numCourses
    for course, pre in prerequisites:
        graph[pre].append(course)
        indegree[course] += 1
    q = deque(i for i in range(numCourses) if indegree[i] == 0)
    taken = 0                            # ← generator in deque!
    while q:
        node = q.popleft()
        taken += 1
        for nei in graph[node]:
            indegree[nei] -= 1
            if indegree[nei] == 0:
                q.append(nei)
    return taken == numCourses

# ── Topological Sort — DFS cycle detection ──
def canFinish_dfs(numCourses, prerequisites):
    graph = defaultdict(list)
    for c, p in prerequisites: graph[p].append(c)
    WHITE, GRAY, BLACK = 0, 1, 2       # ← state enum
    color = [WHITE] * numCourses
    def dfs(node):
        color[node] = GRAY
        for nei in graph[node]:
            if color[nei] == GRAY: return False  # cycle!
            if color[nei] == WHITE and not dfs(nei): return False
        color[node] = BLACK
        return True
    return all(dfs(i) for i in range(numCourses) if color[i] == WHITE)

# • defaultdict(list)   — no KeyError, auto-creates adj list
# • deque(generator)    — initialize queue with generator
# • all(... for ...)    — short-circuit over all nodes
# • WHITE, GRAY, BLACK  — multiple assignment for states

class Trie:
    def __init__(self):
        self.root = {}

    def insert(self, word):
        node = self.root
        for c in word:
            node = node.setdefault(c, {}) # ← create if missing!
        node['#'] = True                  # ← end-of-word marker

    def search(self, word):
        node = self.root
        for c in word:
            if c not in node: return False
            node = node[c]
        return '#' in node

    def startsWith(self, prefix):
        node = self.root
        for c in prefix:
            if c not in node: return False
            node = node[c]
        return True

1️⃣6️⃣ Union Find (Disjoint Set)

# ── Template: Union Find with path compression + rank ──
class UnionFind:
    def __init__(self, n):
        self.par = list(range(n))       # ← parent[i] = i initially
        self.rank = [0] * n

    def find(self, x):
        if self.par[x] != x:
            self.par[x] = self.find(self.par[x])  # ← path compression
        return self.par[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py: return False         # already connected
        if self.rank[px] < self.rank[py]:
            px, py = py, px              # ← swap trick
        self.par[py] = px
        if self.rank[px] == self.rank[py]:
            self.rank[px] += 1
        return True

# ── 1) Kernighan's algorithm — popcount in O(set bits) ──
# Faster than bin(n).count('1') when bits are sparse, and the
# interview-favorite "count 1 bits without bin()" answer.
def popcount(n):
    count = 0
    while n:
        n &= n - 1                       # ← drops the lowest set bit
        count += 1
    return count

# ── 2) Fenwick / Binary Indexed Tree — i & -i is the engine ──
# Used in: Range Sum Query Mutable, Count of Smaller Numbers After Self.
class BIT:
    def __init__(self, n):
        self.tree = [0] * (n + 1)
    def update(self, i, delta):
        while i < len(self.tree):
            self.tree[i] += delta
            i += i & -i                  # ← jump to next responsible node
    def query(self, i):                  # prefix sum [1..i]
        s = 0
        while i > 0:
            s += self.tree[i]
            i -= i & -i                  # ← peel off lowest set bit
        return s

    for i, c in enumerate(s):
        end = max(end, last[c])

            start = i + 1
    return res

# ── 4) heapq.heappushpop / heapreplace — atomic top-K ──
# heappushpop(h, x): push then pop, ONE sift-down (faster than push + pop).
# heapreplace(h, x): pop then push — same cost. Use when h is non-empty.
import heapq
def topK(nums, k):
    h = nums[:k]
    heapq.heapify(h)                    # O(k) min-heap of first k
    for n in nums[k:]:
        if n > h[0]:
            heapq.heapreplace(h, n)     # ← one sift, not two
    return h                            # the top k (unsorted)

# ── Bonus: deque(maxlen=k) — auto-evicting fixed window ──
from collections import deque
window = deque(maxlen=k)                # appending past k auto-pops left
for x in stream:
    window.append(x)                    # window holds at most last k

💡 Why each one

  • Multi-key sort: Java's Comparator.comparing(...).thenComparing(...).reversed() chains collapse to a 1-line tuple key. Negation only works on numerics — for strings, use two passes (Python's sort is stable) or cmp_to_key
  • next(gen, default): O(1) memory, short-circuits on first match. Forgetting the default is the #1 bug — always supply one
  • max(d, key=d.get): argmax in 4 tokens. Ties broken by insertion order (3.7+) — call sorted(d, key=...) if you need a deterministic tiebreak
  • heapreplace vs heappush + heappop: same big-O but ~2× faster in practice — important inside top-K hot loops on large streams
