📐 Syntax Templates

Multi-slot set APIs and common gotchas — labeled syntax plus concrete input → output so you can predict the result before running it.

🏗️ Construct template — set(iterable)

Build a set from any iterable. Duplicates are dropped automatically.

set()                          # → set()             empty set (NOT {})
set([1, 2, 2, 3])               # → {1, 2, 3}         dups dropped
set("hello")                  # → {'h', 'e', 'l', 'o'}   string is iterable
set(range(5))                 # → {0, 1, 2, 3, 4}

# Literal form (when you have known elements):
{1, 2, 3}                       # → {1, 2, 3}

➕ Update template — s.update(*iterables) / s |= other

Bulk add. update can take multiple iterables in one call.

s = {1, 2}

s.update([3, 4])               # s → {1, 2, 3, 4}
s.update([5], (6, 7), {8})       # s → {1, 2, 3, 4, 5, 6, 7, 8}   multiple iterables in one call

# Operator form — same effect, only one iterable:
s = {1, 2}
s |= {3, 4}                     # s → {1, 2, 3, 4}    same as s.update({3, 4})
  • update mutates in place; | (no =) builds a NEW set
  • Sister methods: intersection_update / &=, difference_update / -=, symmetric_difference_update / ^=

📐 Set comprehension template — {expr for x in iterable if cond}

Same shape as a list comprehension; the {} outer braces produce a set instead of a list.

{ what_to_return  for thing in things  if keep_condition }
        ↑                  ↑                    ↑
    expression         iteration         optional filter

{x*x for x in range(5)}            # → {0, 1, 4, 9, 16}    squares
{x for x in [1,2,2,3,3,3] if x > 1}  # → {2, 3}              dedup + filter in one pass
{ch.lower() for ch in "Hello"}        # → {'h', 'e', 'l', 'o'} normalize then dedup

Same template anatomy as the list comprehension — see list.html for the full slot breakdown.

⚖️ Set operators — |, &, -, ^

Four binary operators, four named methods. Operators require both operands to be sets; methods accept any iterable.

a = {1, 2, 3}
b = {2, 3, 4}

a | b      # → {1, 2, 3, 4}    union: in EITHER         a.union(b)
a & b      # → {2, 3}          intersection: in BOTH    a.intersection(b)
a - b      # → {1}             difference: in a NOT b   a.difference(b)
a ^ b      # → {1, 4}          symmetric: in EXACTLY ONE a.symmetric_difference(b)

# Method form takes any iterable; operator form requires both to be sets:
a.union([5, 6])               # → {1, 2, 3, 5, 6}    works on a list
a | [5, 6]                    # → TypeError

⚠️ Gotcha — discard(x) vs remove(x)

Same arg shape, different miss behavior. Pick by whether "missing" is expected.

s = {1, 2, 3}

s.discard(2)         # s → {1, 3}    silent on miss
s.discard(99)        # s → {1, 3}    99 not present, no error

s.remove(3)          # s → {1}       removed
s.remove(99)         # → KeyError: 99    raises if missing

Mental model: discard = "delete if present"; remove = "must be there, crash if not". Same shape as dict get vs [] and string find vs index.

🪤 Gotcha — {} is an empty DICT, not a set

type({})                   # → <class 'dict'>       NOT a set!
type({1})                  # → <class 'set'>        single-element literal IS a set
type(set())                # → <class 'set'>        the only way to get an empty set

See the "Method form & frozenset" tips-box below for operator vs method depth, frozenset hashability, and BFS state-key idioms.

🐍 Python
s = set()                         # empty (NOT {}!)
s = {1, 2, 3}                     # literal
s = set([1, 2, 2, 3])             # from list → {1,2,3}
s = {x for x in range(5)}        # comprehension
☕ Java
Set<Integer> set = new HashSet<>();
Set<Integer> set = new HashSet<>(Arrays.asList(1,2,3));
Set<Integer> set = Set.of(1,2,3); // immutable
// No set comprehension

Operations

🐍 Python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

# ── Add / Remove ──
a.add(7)                    # {1,2,3,4,7}
a.remove(7)                 # KeyError if missing!
a.discard(99)               # safe remove (no error)

# ── Check ──
3 in a                     # True (O(1))

# ── Set Operations (Python superpower!) ──
a | b                       # union → {1,2,3,4,5,6}
a & b                       # intersection → {3,4}
a - b                       # difference → {1,2}
a ^ b                       # symmetric diff → {1,2,5,6}
a <= b                      # is subset?

# ── Iterate ──
for x in a:                 # iterate (unordered)

# ── Convert ──
list(a)                     # set → list
set([1,2,2,3])              # list → set (dedup)

# ── Size ──
len(a)                      # 4
☕ Java
Set<Integer> a = new HashSet<>(Set.of(1,2,3,4));
Set<Integer> b = new HashSet<>(Set.of(3,4,5,6));

// ── Add / Remove ──
a.add(7);                    // {1,2,3,4,7}
a.remove(7);                 // returns false if missing
a.remove(99);                // safe (returns false)

// ── Check ──
a.contains(3);              // true (O(1))

// ── Set Operations (more verbose) ──
Set<Integer> union = new HashSet<>(a);
union.addAll(b);             // union
Set<Integer> inter = new HashSet<>(a);
inter.retainAll(b);          // intersection
Set<Integer> diff = new HashSet<>(a);
diff.removeAll(b);           // difference
// No built-in symmetric diff or subset operator

// ── Iterate ──
for (int x : a) { }         // iterate (unordered)

// ── Convert ──
new ArrayList<>(a);          // set → list
new HashSet<>(list);         // list → set (dedup)

// ── Size ──
a.size();                    // 4

🎯 Method form & frozenset

# ── Operator form vs method form ──
# Operators (|, &, -, ^) require BOTH sides to be sets:
a | b                                # TypeError if b is a list

# Method form takes ANY iterable — handy when b is a list/generator:
a.union(b)                           # works with a.union([1,2,3]) too
a.intersection(b)                    # a.intersection(other_iter)
a.difference(b)                      # a.difference(some_list)
a.symmetric_difference(b)            # elements in exactly one of the two

# Mutating method forms (modify a in place):
a.update(b)                          # a |= b
a.intersection_update(b)             # a &= b
a.difference_update(b)               # a -= b

# ── frozenset — hashable set ──
# Regular sets are unhashable: can't be dict keys or set elements.
# frozenset is the immutable, hashable variant.
fs = frozenset({1, 2, 3})         # or frozenset([1, 2, 3])
seen = set()
seen.add(frozenset(group))         # ✅ works as a set element
cache[frozenset(state)] = result   # ✅ works as a dict key

# ── Common BFS-state use case (LC 864 Shortest Path to Get All Keys) ──
# Visited set keyed by (position, frozenset_of_keys_collected):
visited.add((r, c, frozenset(keys)))

🛠️ Built-ins commonly paired with sets

When you write set code in interviews, these are the built-ins you reach for most.

# ── Membership (the whole point of a set) ──
x in s                                    # O(1) — vs O(n) for list
x not in s

# ── Iterating ──
for x in s: ...                           # unordered!
for x in sorted(s): ...                   # iterate in sorted order

# ── Building ──
{x for x in arr if cond}                  # set comprehension — dedups too
set(arr)                                  # dedup any iterable
set(zip(rows, cols))                     # set of pairs (BFS visited keys)
set(range(n)) - seen                      # remaining via set difference

# ── Aggregating ──
len(s), sum(s), max(s), min(s)
any(x > 0 for x in s)
all(c.isalpha() for c in s)             # works on a set of chars
