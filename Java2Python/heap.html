⬆️ Push/Pop template — heappush(heap, item) / heappop(heap)

A heap is just a list maintained in heap order by these functions. Both ops are O(log n).

import heapq

heapq.heappush( heap , item )
                ↑      ↑
                list   value (any orderable
                kept   type — int, tuple,
                in     etc.)
                heap
                order

h = []
heapq.heappush(h, 5)
heapq.heappush(h, 1)
heapq.heappush(h, 3)        # h is now [1, 3, 5] (heap-ordered, smallest at h[0])

heapq.heappop(h)              # → 1   removes & returns smallest
heapq.heappop(h)              # → 3
h[0]                          # peek smallest WITHOUT popping (cheap)
  • Always use heapq.heappush / heapq.heappop — never list.append / list.pop on a heap (breaks the invariant)
  • h[0] is the smallest element; that's the only random-access guarantee. The rest is heap-ordered, NOT sorted

🔀 Combined-op template — heappushpop / heapreplace

One log-n traversal instead of two. Different orderings — pick by what should always go in.

heapq.heappushpop( heap , item )    # PUSH first, then POP smallest
heapq.heapreplace( heap , item )    # POP smallest first, then PUSH (item ALWAYS goes in)

h = [1, 3, 5]
heapq.heappushpop(h, 0)        # → 0    0 is smaller than 1, so 0 pushes-then-pops itself out; h unchanged
heapq.heappushpop(h, 4)        # → 1    4 pushed, then 1 popped (was smallest); h = [3, 4, 5]

h = [1, 3, 5]
heapq.heapreplace(h, 0)        # → 1    1 popped first, THEN 0 pushed; h = [0, 3, 5]
heapq.heapreplace(h, 4)        # → 0    0 popped first, THEN 4 pushed; h = [3, 4, 5]
  • heappushpop: the smaller of item and h[0] comes out. Use when "either item could be the new minimum"
  • heapreplace: h[0] always pops; item always goes in. Use when you know item belongs (e.g. streaming top-K, where every new item is a candidate)

🏆 Top-K template — nlargest(n, iterable, key=None) / nsmallest(n, iterable, key=None)

Three slots: how many, what to scan, and an optional rank function. Uses O(n) memory (a heap of size n), not O(len(iterable)).

heapq.nlargest( n , iterable , key=None )
                ↑    ↑          ↑
                how  what to    optional rank
                many scan       function
                                (called per item)

heapq.nlargest(3, [5, 1, 3, 8, 2])           # → [8, 5, 3]      sorted descending
heapq.nsmallest(3, [5, 1, 3, 8, 2])          # → [1, 2, 3]      sorted ascending

# With key= function:
words = ["fig", "banana", "pear", "apple"]
heapq.nlargest(2, words, key=len)            # → ['banana', 'apple']  longest two

# Top-K by frequency from a Counter:
from collections import Counter
c = Counter("abracadabra")
heapq.nlargest(3, c, key=c.get)            # → ['a', 'b', 'r']     top-3 most common keys
  • For top-K from a stream of unknown length, this is the right tool — it doesn't materialize the full input
  • For n == 1, prefer max/min (clearer); for n >= len/2, just sorted(...)[:n]
  • key= works the same as in sorted — function called on each item; ranking uses the return value

🏗️ Heapify template — heapq.heapify(list)

In-place, O(n) — faster than pushing items one at a time (which would be O(n log n)).

arr = [5, 1, 3, 8, 2]
heapq.heapify(arr)            # mutates arr → [1, 2, 3, 8, 5]   returns None!
arr[0]                        # → 1    smallest is now at index 0

# ⚠️ TRAP — heapify returns None:
h = heapq.heapify(arr)        # h is None! arr was mutated in place
# Just call it:
heapq.heapify(arr)            # correct — arr is now a heap

🪢 Merge template — heapq.merge(*iterables, key=None, reverse=False)

Lazy merge of pre-sorted iterables. Returns an iterator — wrap in list(...) to materialize.

a = [1, 4, 7]
b = [2, 3, 9]
c = [5, 6, 8]

list(heapq.merge(a, b, c))            # → [1, 2, 3, 4, 5, 6, 7, 8, 9]
list(heapq.merge(a, b, reverse=True))   # inputs must be DESC if reverse=True
  • Inputs MUST already be sorted (ascending by default); merge doesn't sort
  • Lazy: useful for merging huge sorted files without loading them fully

⚠️ Gotcha — heapq is a MIN-heap (use negation for max-heap)

# Max-heap of ints — push negated, negate on pop:
h = []
for x in [5, 1, 8, 3]:
    heapq.heappush(h, -x)
-heapq.heappop(h)             # → 8   the original max

# Max-heap of (priority, payload) tuples — negate the priority only:
heapq.heappush(h, (-priority, payload))

🎯 Gotcha — tuple priorities need a tiebreaker

Python compares tuples element-wise. If two priorities tie and the payload isn't comparable, you'll get a TypeError.

# ❌ Will crash if two priorities are equal AND payload isn't orderable:
heapq.heappush(h, (priority, payload))

# ✅ Add a unique tiebreaker (insertion counter) to break ties cheaply:
import itertools
counter = itertools.count()
heapq.heappush(h, (priority, next(counter), payload))
# Tuples compare priority first; if equal, the (always-unique) counter wins.

See the existing tips-box below for nlargest/nsmallest use with Counter and other patterns.

Operations

🐍 Python
import heapq

# ── Min Heap ──
heap = []
heapq.heappush(heap, 5)      # [5]
heapq.heappush(heap, 1)      # [1,5]
heapq.heappush(heap, 3)      # [1,5,3]
heap[0]                      # peek min → 1
heapq.heappop(heap)          # pop min → 1

# ── Max Heap (negate values!) ──
heapq.heappush(heap, -val)
-heapq.heappop(heap)

# ── Heapify existing list ──
nums = [5, 3, 8, 1]
heapq.heapify(nums)          # O(n), in-place

# ── Push + Pop combined ──
heapq.heappushpop(heap, val) # push then pop min
heapq.heapreplace(heap, val) # pop min then push

# ── Top K Pattern ──
heapq.nlargest(3, nums)      # 3 largest
heapq.nsmallest(3, nums)     # 3 smallest

# ── With tuples (priority, data) ──
heapq.heappush(heap, (2, "low"))
heapq.heappush(heap, (1, "high"))
heapq.heappop(heap)          # (1, "high")
☕ Java
// ── Min Heap ──
PriorityQueue<Integer> pq = new PriorityQueue<>();
pq.offer(5);                 // [5]
pq.offer(1);                 // [1,5]
pq.offer(3);                 // [1,5,3]
pq.peek();                   // peek min → 1
pq.poll();                   // pop min → 1

// ── Max Heap (pass comparator!) ──
PriorityQueue<Integer> maxPQ =
    new PriorityQueue<>(Collections.reverseOrder());

// ── Heapify existing collection ──
PriorityQueue<Integer> pq =
    new PriorityQueue<>(Arrays.asList(5,3,8,1));

// ── No push+pop combo ──
// Must do manually: pq.offer(val); pq.poll();

// ── No built-in nlargest/nsmallest ──
// Use stream: nums.stream().sorted()
//   .limit(3).toList();

// ── With custom objects ──
PriorityQueue<int[]> pq = new PriorityQueue<>(
    (a,b) -> a[0] - b[0]);     // sort by first elem
pq.offer(new int[]{2, 0});  // {priority, data}
pq.offer(new int[]{1, 1});
pq.poll();                   // [1, 1]

🛠️ Built-ins commonly paired with heaps

When you write heap code in interviews, these are the built-ins you reach for most.

import heapq

# ── Top-K shortcuts (bypass heappush/heappop loops) ──
heapq.nlargest(k, arr)                    # top k as list
heapq.nsmallest(k, arr)
heapq.nlargest(k, items, key=lambda x: x.score)  # objects by attribute

# ── Tuples as priorities ──
heapq.heappush(pq, (priority, tiebreaker, payload))
val, _, obj = heapq.heappop(pq)           # tuple unpacking on pop

# ── Heapify in place (O(n) — faster than n × heappush) ──
heapq.heapify(arr)

# ── Counter + heapq combo (top-k frequent) ──
from collections import Counter
count = Counter(arr)
heapq.nlargest(k, count, key=count.get)   # keys with top-k counts

# ── Iterating: a heap is just a list ──
for x in pq: ...                         # NOT in sorted order — array layout
for x in sorted(pq): ...                 # sorted snapshot (allocates)
len(pq), pq[0]                          # size, peek-min — no method call needed

# ── Max-heap via negation ──
heapq.heappush(pq, -val)
-heapq.heappop(pq)                        # flip sign back on extract

Full reference for any of these: Python Builtins.

Interview Pattern: Heap / Top-K / Two Heaps

Priority queue for K-th element, streaming median. Used in: Kth Largest Element, Top K Frequent, Merge K Sorted Lists, Find Median from Data Stream, Task Scheduler

🐍 Python Shortcuts
import heapq

# ── Top K Frequent Elements (LC 347) ──
def topKFrequent(nums, k):
    count = Counter(nums)                # ← instant frequency
    return heapq.nlargest(k, count, key=count.get)  # ← one-liner!
    # Alt: return [x for x, _ in count.most_common(k)]

# ── Kth Largest Element (LC 215) ──
def findKthLargest(nums, k):
    return heapq.nlargest(k, nums)[-1]   # ← built-in!

# ── Merge K Sorted Lists (LC 23) ──
def mergeKLists(lists):
    heap = []
    for i, l in enumerate(lists):
        if l:
            heapq.heappush(heap, (l.val, i, l))  # ← i breaks ties!
    dummy = cur = ListNode(0)
    while heap:
        val, i, node = heapq.heappop(heap)
        cur.next = node
        cur = cur.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next

# ── Find Median (LC 295) — Two Heaps ──
# small = max-heap (negate!), large = min-heap
small, large = [], []  # max-heap (neg), min-heap
heapq.heappush(small, -num)
heapq.heappush(large, -heapq.heappop(small))

# ── Python tricks used ──
# • heapq.nlargest(k, ...)  — avoid full sort
# • Counter().most_common(k) — top K frequencies
# • -val for max-heap       — negate trick
# • (val, tiebreaker, obj)  — tuple comparison order
