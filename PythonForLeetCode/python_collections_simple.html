add or update?   d[k] = v is the same syntax for both. If k is new it adds; if k exists it overwrites. Python doesn't distinguish.
defaultdict differs   reading a missing key auto-creates the default and returns it — no KeyError. Factory examples: defaultdict(int) → 0 (counting), defaultdict(list) → [] (grouping), defaultdict(set) → set(). So dd[k] += 1 and dd[k].append(x) work even on first touch. Everything else: identical to dict.

frozenset differs   immutable — no .add, .discard, .remove, .pop, .update, .clear. Same reads (in, len, | & - ^). Hashable — usable as a dict key or inside another set.

max-heap trick   push -x instead of x, negate again on pop. For object priorities: push tuples (priority, item); smallest priority comes out first.
no arbitrary remove   heapq can only remove the smallest in O(log n). Removing some specific element x is O(n) — you'd list.remove(x) then heapify. If you need O(log n) arbitrary remove, use SortedList.
