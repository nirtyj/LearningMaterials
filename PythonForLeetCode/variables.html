Variables & Memory Model

References, immutability, and the copy traps that bite you

# Everything is an object reference
x = 42          # x references int object
y = x           # y references SAME object
x = 100         # x now references NEW object

# No type declarations needed
name = "Alice"  # str
age = 25        # int
pi = 3.14       # float
is_valid = True # bool

# Dynamic typing
x = 10          # int
x = "hello"     # now str (allowed!)
Python Memory

Every variable is a reference to a heap object — including int and bool. There are no "primitives" on a stack.

x → int(42) @ 0x7f...
name → str("Alice") @ 0x8a...
nums → list([1,2,3]) @ 0x9b...

💡 Interview Insight: Mutability

  • Immutable: int, float, str, tuple, frozenset — "mutating" returns a new object.
  • Mutable: list, dict, set — methods like append, pop, update change the object in place.
  • a = b on a list creates a reference, not a copy. a.append(x) mutates b too.
  • Shallow copy: a = b.copy() or a = b[:]. Deep copy (nested lists): copy.deepcopy(b).
  • Mutable default args trap: def f(xs=[]): — the list is shared across calls. Use xs=None + xs = xs or [].
