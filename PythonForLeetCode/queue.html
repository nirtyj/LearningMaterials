📐 Syntax Templates

Multi-slot deque APIs and common gotchas — labeled syntax plus concrete input → output so you can predict the result before running it.

🏗️ Construct template — deque(iterable=(), maxlen=None)

Two slots: initial contents and an optional bound. A bounded deque drops from the opposite end on overflow.

from collections import deque

deque( iterable , maxlen )
        ↑          ↑
        initial    optional bound
        contents   (default None = unbounded)
        (default
        empty)

deque()                          # → deque([])              empty
deque([1, 2, 3])                  # → deque([1, 2, 3])       from any iterable
deque("abc")                     # → deque(['a', 'b', 'c']) string is iterable too
deque([1, 2, 3], maxlen=2)         # → deque([2, 3], maxlen=2) front item dropped to fit
  • maxlen-bounded deque: pushing on one end pops from the other when full — useful for sliding windows and "last N items" caches
  • Pass any iterable as the first arg — list, tuple, string, generator

↔️ Both-ends template — append / appendleft / pop / popleft

All four are O(1). The "left" variants are why deque exists — list.pop(0) is O(n).

dq = deque([1, 2, 3])

dq.append(4)              # → deque([1, 2, 3, 4])    add right (the "queue tail")
dq.appendleft(0)          # → deque([0, 1, 2, 3, 4]) add left (the "queue head")

dq.pop()                  # → 4   removes & returns RIGHTMOST
dq.popleft()              # → 0   removes & returns LEFTMOST

# BFS pattern (the canonical use):
queue = deque([start])
while queue:
    node = queue.popleft()        # O(1) dequeue from head
    for nbr in graph[node]:
        queue.append(nbr)         # O(1) enqueue at tail
  • Mental model: append/pop = right end (tail); appendleft/popleft = left end (head)
  • FIFO queue: append + popleft. Stack on a deque: append + pop. LIFO from the left: appendleft + popleft.

📥 Bulk template — extend(iterable) / extendleft(iterable)

Add many items at once. extendleft has a non-obvious quirk — see the gotcha below.

dq = deque([1])

dq.extend([2, 3, 4])
# → deque([1, 2, 3, 4])      items appended in order

dq = deque([1])
dq.extendleft([2, 3, 4])
# → deque([4, 3, 2, 1])      items appendleft'd one at a time → REVERSED!

🔄 Rotate template — dq.rotate(n=1)

Circular shift. Positive shifts right (last items wrap to front); negative shifts left.

dq = deque([1, 2, 3, 4, 5])

dq.rotate(1)                # → deque([5, 1, 2, 3, 4])  last → front (right by 1)
dq.rotate(2)                # → deque([4, 5, 1, 2, 3])  right by 2 (cumulative from above)

dq = deque([1, 2, 3, 4, 5])
dq.rotate(-1)               # → deque([2, 3, 4, 5, 1])  first → back (left by 1)
  • Positive n: rightward shift (think "rotate clockwise" — items roll off the right and onto the left)
  • Negative n: leftward shift
  • Useful for round-robin scheduling, circular buffers, and the "rotate array by k" interview question

⚠️ Gotcha — extendleft reverses

Each item in the iterable is appendleft'd in turn — so the LAST item ends up leftmost.

dq = deque()
dq.extendleft([1, 2, 3])      # → deque([3, 2, 1])   not [1, 2, 3]!

# Equivalent to:
for x in [1, 2, 3]:
    dq.appendleft(x)             # 1 lands first → pushed left by 2 → pushed left by 3

# Want left-prepend in original order? Reverse the input:
dq.extendleft(reversed([1, 2, 3]))
# → deque([1, 2, 3])

🪤 Gotcha — maxlen drops from the OPPOSITE end

Pushing right when full pops left, and vice versa. That's the sliding-window superpower.

dq = deque(maxlen=3)
dq.append(1); dq.append(2); dq.append(3)
# → deque([1, 2, 3], maxlen=3)

dq.append(4)                  # → deque([2, 3, 4], maxlen=3)   pushed right, 1 popped from left
dq.appendleft(0)              # → deque([0, 2, 3], maxlen=3)   pushed left, 4 popped from right

Operations

from collections import deque

q = deque()

# ── Enqueue / Dequeue ──
q.append(1)                  # right → deque([1])
q.append(2)                  # right → deque([1,2])
q.appendleft(0)              # left  → deque([0,1,2])
q.popleft()                  # dequeue → 0 (O(1)!)
q.pop()                      # from right → 2

# ── Peek ──
q[0]                         # front
q[-1]                        # back

🛠️ Built-ins commonly paired with queues / deques

When you write queue / deque code in interviews, these are the built-ins you reach for most.

from collections import deque

# ── Building ──
q = deque()
q = deque([root])                         # init with single element (BFS start)
q = deque(arr)                            # init from any iterable
q = deque(maxlen=k)                       # bounded — auto-pops oldest

# ── BFS level-snapshot pattern (the universal trick) ──
for _ in range(len(q)):                   # snapshot size BEFORE iterating
    node = q.popleft()
    q.extend(node.children)               # bulk append — extend = append-many

# ── Iterating without consuming ──
for x in q: ...                          # left-to-right
list(q)                                   # snapshot copy
enumerate(q)                              # index + value

# ── Index-style peek ──
q[0]                                       # front
q[-1]                                      # back

# ── Sliding window with maxlen ──
window = deque(maxlen=k)
for x in stream:
    window.append(x)                      # window auto-trims to last k items
    if len(window) == k: print(sum(window))

Interview Pattern: BFS (Level-Order / Shortest Path)

Queue-based level traversal for trees / graphs. Used in: Level Order Traversal, Rotting Oranges, Word Ladder, Walls and Gates, Number of Islands

🐍 Python Shortcuts
# ── Template: Tree level-order BFS ──
from collections import deque
def levelOrder(root):
    if not root: return []
    res, q = [], deque([root])
    while q:
        level = []
        for _ in range(len(q)):         # ← process one level
            node = q.popleft()
            level.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        res.append(level)
    return res

# ── Template: Graph/Grid BFS ──
def bfs(grid, start):
    q = deque([start])
    visited = {start}                    # ← set for O(1) lookup
    steps = 0
    while q:
        for _ in range(len(q)):
            r, c = q.popleft()           # ← tuple unpacking
            for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:  # ← 4 directions
                nr, nc = r + dr, c + dc
                if 0 <= nr < rows and 0 <= nc < cols \
                   and (nr, nc) not in visited:
                    visited.add((nr, nc)) # ← tuple as set element
                    q.append((nr, nc))
        steps += 1
    return steps

# ── Python tricks used ──
# • deque([start])      — O(1) popleft (list.pop(0) is O(n)!)
# • for _ in range(len(q))  — snapshot level size
# • (r, c) tuples       — hashable, can go in sets
# • 0 <= nr < rows      — chained comparison
