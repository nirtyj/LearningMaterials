📊 DefaultDict / Counter

Python's frequency & grouping superpowers

📎 Companion refs: Dict / HashMap (when plain dict suffices) · Python Builtins → ord/chr · Heap (Top-K from a Counter)

Initialization

📌 For one-off counting on small inputs, plain dict.get(k, 0) + 1 is fine — see Dict / HashMap → Frequency pattern. Counter shines when you need .most_common(n), multiset arithmetic (+ - & |), or zero-default lookups without the .get ceremony.

📐 Syntax Templates

Multi-slot Counter APIs and common gotchas — labeled syntax plus concrete input → output so you can predict the result before running it.

🏗️ Construct template — three forms

Counter accepts three constructor shapes: an iterable, a mapping, or keyword args.

from collections import Counter

Counter( iterable | mapping | **kwargs )

# 1) From an iterable — counts each element:
Counter("banana")              # → Counter({'a': 3, 'n': 2, 'b': 1})
Counter([1, 2, 2, 3, 3, 3])      # → Counter({3: 3, 2: 2, 1: 1})

# 2) From a mapping (existing counts):
Counter({"a": 2, "b": 1})         # → Counter({'a': 2, 'b': 1})

# 3) From keyword args:
Counter(a=2, b=1)              # → Counter({'a': 2, 'b': 1})

# Empty Counter (any missing key returns 0 — that's the whole point):
c = Counter()
c["x"]                        # → 0      no KeyError, no insertion

🏆 Most-common template — c.most_common(n=None)

Returns a list of (element, count) pairs in count-descending order.

c = Counter("aaabbc")              # Counter({'a': 3, 'b': 2, 'c': 1})

c.most_common(2)                   # → [('a', 3), ('b', 2)]      top 2
c.most_common(1)                   # → [('a', 3)]                top 1
c.most_common()                    # → [('a', 3), ('b', 2), ('c', 1)]   no arg = all sorted

# For just the top key (not the count), index into it:
c.most_common(1)[0][0]             # → 'a'    top element only
  • Returns [(elem, count), ...] — note it's a list of tuples, not a Counter
  • For top-K with a custom rank, prefer heapq.nlargest(n, c, key=c.get) — see heap.html

➕ Update / subtract template — c.update(iter) / c.subtract(iter)

Bulk additive update. subtract can drive counts negative (unlike the - operator).

c = Counter("abc")               # Counter({'a': 1, 'b': 1, 'c': 1})

c.update("aab")                  # Counter({'a': 3, 'b': 2, 'c': 1})   ADDS counts
c.update({"d": 5})              # Counter({'d': 5, 'a': 3, 'b': 2, 'c': 1})   accepts mapping too

c = Counter("abc")
c.subtract("aab")                # Counter({'c': 1, 'b': 0, 'a': -1})  ALLOWS negatives/zero
  • update ADDS to existing counts (it does NOT replace, unlike dict.update which would overwrite)
  • subtract permits negative results — use when you need to track surplus/deficit

➗ Counter arithmetic — +, -, &, |

Multiset arithmetic. The four operators have different semantics from set operators!

c1 = Counter("aab")              # {'a': 2, 'b': 1}
c2 = Counter("abb")              # {'a': 1, 'b': 2}

c1 + c2                          # → Counter({'a': 3, 'b': 3})       ADD counts
c1 - c2                          # → Counter({'a': 1})               SUBTRACT, drop ≤0 (b: 1-2=-1 dropped)
c1 & c2                          # → Counter({'a': 1, 'b': 1})       MIN of each (intersection-like)
c1 | c2                          # → Counter({'a': 2, 'b': 2})       MAX of each (union-like)
  • & = elementwise min, | = elementwise max — multiset semantics, NOT set intersection/union
  • Binary operators (+/-) DROP zero and negative counts. c1.subtract(c2) KEEPS them — see template above

✨ Gotcha — missing keys return 0 (not KeyError)

c = Counter("abc")
c["a"]                          # → 1      present
c["x"]                          # → 0      missing — returns 0, NO insertion, NO exception

# That's the entire point of Counter — count without guarding for KeyError.
# Compare to dict[k] which raises, or dict.get(k, 0) which is verbose.

# ⚠️ Side-note: this is a "lookup", not "insertion":
"x" in c                       # → False    accessing c["x"] does NOT add it

🔀 Gotcha — Counter &/| ≠ set &/|

Same operators, different semantics. Easy to confuse if you've been working with sets.

# Sets — boolean membership:
{1, 2, 3} & {2, 3, 4}              # → {2, 3}      INTERSECTION
{1, 2, 3} | {2, 3, 4}              # → {1, 2, 3, 4}  UNION

# Counters — multiset (counts):
Counter("aab") & Counter("abb")       # → Counter({'a': 1, 'b': 1})  MIN of each count
Counter("aab") | Counter("abb")       # → Counter({'a': 2, 'b': 2})  MAX of each count

# ── DefaultDict ──
from collections import defaultdict
dd = defaultdict(int)              # default 0
dd = defaultdict(list)             # default []
dd = defaultdict(lambda: -1)       # default -1

# ── Counter ──
from collections import Counter
c = Counter([1,1,2,3])            # from list
c = Counter("hello")              # from string
c = Counter({"a":2, "b":1})       # from dict

# ── Tuple (immutable, hashable, multiset-key) ──
t = ()                             # empty
t = (1, 2, 3)                     # literal
t = (1,)                          # single elem (need comma!)

Operations

from collections import defaultdict, Counter

# ── DefaultDict (auto-creates missing keys) ──

# Group anagrams pattern
groups = defaultdict(list)
for word in ["eat","tea","tan","ate","nat"]:
    key = tuple(sorted(word))
    groups[key].append(word)
# {('a','e','t'): ['eat','tea','ate'],
#  ('a','n','t'): ['tan','nat']}

# Graph adjacency list
graph = defaultdict(list)
for u, v in edges:
    graph[u].append(v)
    graph[v].append(u)         # undirected

# Counting with defaultdict
counts = defaultdict(int)
for x in data:
    counts[x] += 1             # no KeyError!

# ── Counter ──
c = Counter("abracadabra")
c.most_common(3)             # [('a',5),('b',2),('r',2)]
c["a"]                       # 5
c["z"]                       # 0 (no KeyError!)

# Counter arithmetic
c1 = Counter("aab")
c2 = Counter("ab")
c1 - c2                      # Counter({'a': 1})
c1 & c2                      # intersection: min of each
c1 | c2                      # union: max of each

Anagram & Multiset Keys

🔤 ord, tuple(sorted(s)), Counter arithmetic

The toolkit for any "are these the same multiset?" problem: Group Anagrams, Valid Anagram, Find All Anagrams in a String, Minimum Window Substring, Permutation in String, Ransom Note. 

# ── 1) ord(c) - ord('a') — letter → 0..25 bucket ──
# Python has no implicit char→int — see python-builtins.html#builtins-functions for ord/chr foundations.
count = [0] * 26
for c in word:
    count[ord(c) - ord('a')] += 1

# Inverse: chr(ord('a') + idx)  →  letter from bucket

# ── 2) Two canonical multiset keys for grouping ──
# (a) Sort the word — O(k log k) per word, simplest:
key = tuple(sorted(word))           # ('a','e','t')  hashable!

# (b) 26-int count vector — O(k) per word, faster on long inputs:
count = [0] * 26
for c in word: count[ord(c) - ord('a')] += 1
key = tuple(count)                  # tuples hash; lists don't

# ── 3) Counter compare for "is anagram?" ──
Counter(s) == Counter(t)            # True iff anagrams (LC 242)

# ── 4) Counter.elements() — flatten back to a multiset stream ──
list(Counter("aab").elements())    # ['a','a','b']
# Useful: rebuild a sequence after Counter arithmetic, or feed
# into sorted()/heap operations.

# ── 5) Sliding-window anagram check (LC 438) ──
# Compare two Counters in O(26) — Counter equality is set-based.
need = Counter(p)
window = Counter(s[:len(p)])
if window == need: ans.append(0)
for i in range(len(p), len(s)):
    window[s[i]] += 1
    window[s[i - len(p)]] -= 1
    if window[s[i - len(p)]] == 0:
        del window[s[i - len(p)]]   # ← keep == comparison clean
    if window == need: ans.append(i - len(p) + 1)

# ── 6) dict.fromkeys(arr) — order-preserving dedup ──
# Dicts preserve insertion order (3.7+), so fromkeys + list dedups while keeping first-seen order.
list(dict.fromkeys([3,1,3,2,1]))  # [3, 1, 2]

⚠️ Gotchas

  • list is unhashable → can't use list as dict key. Always wrap a frequency vector in tuple(...) before using it as a key
  • When comparing Counters with ==, delete zero-count keys (or use +window to drop them) — otherwise Counter({'a':1, 'b':0}) != Counter({'a':1})
  • ord('a') == 97, ord('A') == 65 — for mixed case, normalize with .lower() first or use a 52-slot vector
  • Sorted-tuple key is O(k log k) per word but reads cleanest; count-vector key is O(k) — pick by input size, not aesthetics

🛠️ Built-ins commonly paired with Counter / DefaultDict

When you write Counter / DefaultDict code in interviews, these are the built-ins you reach for most.

from collections import Counter

# ── Constructing ──
c = Counter(arr)                          # any iterable
c = Counter("hello")                      # works on strings
c = Counter({"a": 2, "b": 1})             # from a frequency dict

# ── Top-N / sort ──
c.most_common(3)                          # [('a',5), ('b',2), ('c',1)]
sorted(c, key=c.get, reverse=True)        # keys ordered by count desc

# ── Comparing & set-style ops ──
Counter(s) == Counter(t)                  # multisets equal? (anagram check)
c1 + c2                                   # element-wise add
c1 - c2                                   # element-wise subtract (drops zeros & negatives)
c1 & c2                                   # min of each (multiset intersection)
c1 | c2                                   # max of each (multiset union)

# ── Iterating ──
for elem, cnt in c.items(): ...
for elem in c.elements(): ...             # repeats each elem cnt times
sum(c.values())                           # total count

# ── Combining with sum / max / any ──
total_vowels = sum(c[v] for v in "aeiou")
most_common_letter = max(c, key=c.get)
