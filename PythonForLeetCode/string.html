📐 Syntax Templates

Multi-slot string APIs and common gotchas — labeled syntax plus concrete input → output so you can predict the result before running it.

🔪 Slicing — s[start:stop:step]

Strings slice with the same [start:stop:step] grammar as lists — see List → Slice template for the full slot diagram, negative-index rules, and step-sign behavior. Two string-specific things to know:

"hello"[::-1]                       # → "olleh"  idiomatic string reverse
s[0] = "H"                          # ❌ TypeError — strings are immutable
s = "H" + s[1:]                     # ✅ build a new string instead
  • Slices always return a new string (strings are immutable — there is no in-place form)
  • No assignment-by-index: s[i] = c raises TypeError. For repeated character changes, convert to a list, mutate, then "".join(...) — see the Immutability section below

✂️ Split template — s.split(sep=None, maxsplit=-1)

Two slots: what to split on, and the maximum number of splits to make.

s.split( sep , maxsplit )
          ↑      ↑
          delim  stop after N splits
          (None  (-1 = unlimited)
          = any
          whitespace,
          collapsed)

# ── Default sep (None): splits on whitespace runs, trims edges ──
"  a   b  ".split()         # → ['a', 'b']         (collapses runs, trims)
"hello world".split()       # → ['hello', 'world']

# ── Explicit sep ──
"a,b,c".split(",")          # → ['a', 'b', 'c']
"a,b,c".split(",", 1)       # → ['a', 'b,c']       maxsplit=1: stop after first split
"a,,b".split(",")           # → ['a', '', 'b']     empty strings KEPT with explicit sep

# ── Companion: rsplit splits from the right ──
"a,b,c".rsplit(",", 1)      # → ['a,b', 'c']       useful for "split off extension"
  • split() (no arg) is special: splits on ANY whitespace run AND trims leading/trailing whitespace
  • split(" ") (explicit space) is different: keeps empty strings between consecutive spaces
  • maxsplit caps the result at maxsplit + 1 elements — useful when only the first separator matters

🔁 Replace template — s.replace(old, new, count=-1)

Returns a NEW string (strings are immutable). Three slots: what to find, what to replace with, how many times.

s.replace( old , new , count )
            ↑     ↑     ↑
            find  swap  max replacements
                  with  (-1 = all, default)

"banana".replace("a", "X")        # → "bXnXnX"  all occurrences
"banana".replace("a", "X", 1)     # → "bXnana"  first only
"banana".replace("a", "X", 2)     # → "bXnXna"  first two
"banana".replace("z", "X")        # → "banana"  no matches → unchanged copy

# ⚠️ TRAP — without assignment, the new string is discarded:
s = "banana"
s.replace("a", "X")               # returns "bXnXnX" but s is still "banana"!
s = s.replace("a", "X")           # correct: rebind s
  • Always assign the result — strings are immutable; replace never mutates
  • count caps total replacements; use 1 when only the first hit matters
  • For pattern-based replacement, reach for re.sub(pattern, repl, string, count=0)

🔍 Find template — s.find(sub, start=0, end=len(s))

Three slots: what to look for, and an optional substring window. Returns the index, or -1 if not found.

s.find( sub , start , end )
         ↑     ↑       ↑
         find  search  search
         this  from    until
               here    here (exclusive)
               (def 0) (def len(s))

"banana".find("na")              # → 2     first match
"banana".find("na", 3)           # → 4     skip past index 3, find next
"banana".find("na", 3, 5)        # → -1    window too tight to contain "na"
"banana".find("z")               # → -1    not found (no exception)

# ── rfind searches from the right ──
"banana".rfind("na")             # → 4     last occurrence
  • find returns -1 on miss; index raises ValueError — same arg shape, different miss behavior (see Gotcha below)
  • start/end let you search a substring window without slicing first (no allocation)
  • Membership-only check? Use "na" in s — clearer than find >= 0

🪢 Join template — sep.join(iterable)

Glue strings together with a separator. The separator is the THING that calls .join — feels backwards at first, gets natural with use.

sep.join( iterable )
   ↑       ↑
   glue    strings to glue
   string  (must be str — not int!)

",".join(["a", "b", "c"])         # → "a,b,c"
"".join(["h", "i"])               # → "hi"          empty separator
"".join(reversed("hi"))           # → "ih"          works on any iterable of str
"-".join(str(x) for x in [1,2,3])  # → "1-2-3"       cast non-strings first

# ⚠️ TypeError if any element isn't a string:
",".join([1, 2, 3])               # → TypeError: sequence item 0: expected str, int found
  • "".join(parts) is the standard way to build a string from chars/pieces — much faster than repeated += on long inputs
  • All elements must be str — wrap with str(x) or use a generator expression to cast
  • The separator can be empty ("") or multi-char (", ")

⚠️ Gotcha — find vs index

Same arg shape, different miss behavior. Choose by whether "missing" is expected or exceptional.

"banana".find("z")     # → -1                 safe sentinel
"banana".index("z")    # → ValueError: substring not found

# Use find when missing is normal:
i = s.find("@")
if i >= 0: ...

# Use index when missing is a bug (let it crash):
i = s.index("@")            # trust the input had an @

🧊 Gotcha — strings are IMMUTABLE

Every "modifying" method actually returns a new string. Without assignment, the result is thrown away.

s = "hello"
s.upper()                    # returns "HELLO" — but s is still "hello"
s = s.upper()                # now s is "HELLO"

# ❌ Can't assign by index:
s[0] = "H"                 # → TypeError: 'str' object does not support item assignment

# ✅ Build via list, then join:
chars = list(s)              # ['h', 'e', 'l', 'l', 'o']
chars[0] = "H"
s = "".join(chars)          # "Hello"

See Immutability — Why Building With list Matters below for the full pattern.

s = "hello world"

# ── Access ──
s[0]                         # 'h'
s[-1]                        # 'd'
s[1:5]                       # 'ello' (slicing)

# ── Modify (strings are immutable!) ──
s.upper()                    # 'HELLO WORLD'
s.lower()                    # 'hello world'
s.strip()                    # trim whitespace
s.replace("hello", "hi")    # 'hi world'

# ── Split / Join ──
s.split()                    # ['hello','world']
s.split(",")                 # split on comma
" ".join(["a", "b"])        # "a b"

# ── Check ──
s.startswith("he")          # True
s.endswith("ld")             # True
s.isdigit()                  # False
s.isalpha()                  # False (has space)
"ell" in s                  # True (substring check)
s.find("lo")                # 3 (-1 if not found)

# ── Reverse ──
s[::-1]                      # 'dlrow olleh'

# ── Build string efficiently ──
chars = []
chars.append("h")
chars.append("i")
result = "".join(chars)      # "hi"

# ── f-strings ──
name = "Bob"
f"Hello {name}, 2+2={2+2}"  # "Hello Bob, 2+2=4"

🛠️ Built-ins commonly paired with strings

When you write string code in interviews, these are the built-ins you reach for most.

# ── Char access ──
s[0], s[-1], s[i:j]              # indexing, slicing — strings index like lists
ord(c), chr(97)                  # char ↔ int; see python-builtins.html#builtins-functions for the bucket trick

# ── Char checks (per-char or whole-string) ──
c.isalnum(), c.isalpha()         # letter or digit / letter
c.isdigit(), c.isspace()         # digit / whitespace
c.islower(), c.isupper()         # case
c.lower(), c.upper(), c.casefold()  # case-fold for compares

# ── Iterating ──
for c in s: ...                  # direct char iteration
for i, c in enumerate(s): ...    # index + char
for a, b in zip(s, s[1:]): ...   # adjacent pairs (palindrome scans, runs)
for a, b in pairwise(s): ...     # 3.10+ — same idea via itertools

# ── Transforming ──
"".join(reversed(s))             # reverse without slicing
"".join(c for c in s if c.isalnum())  # filter chars in one shot
[c.lower() for c in s if c.isalnum()] # cleaned list of chars

# ── Searching / aggregating ──
s.count("ab")                    # non-overlapping count
sum(1 for c in s if c.isdigit()) # count by predicate
max(s.split(), key=len)          # longest word

Full reference for any of these: Python Builtins.

Immutability — Why Building With list Matters

String Immutability

# Strings are immutable in BOTH languages
s = "hello"
# s[0] = "H"  # ERROR!

# ✅ Create new string
s = "H" + s[1:]

# For many modifications, use list
chars = list(s)
chars[0] = "H"
s = "".join(chars)

Repeated s += "x" in a loop is O(n²) in Python — every += allocates a fresh string. Build a list and "".join() at the end for O(n).
