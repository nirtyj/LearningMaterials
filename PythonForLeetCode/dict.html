📐 Syntax Templates

Multi-slot dict APIs and common gotchas — labeled syntax plus concrete input → output so you can predict the result before running it.

🛡️ Get template — d.get(key, default=None)

Two slots: the key, and what to return when it's missing. Never raises.

d.get( key , default )
        ↑     ↑
        look  returned if key
        up    is missing
              (default None)

d = {"a": 1, "b": 2}

d.get("a")              # → 1            present
d.get("x")              # → None         missing, no exception
d.get("x", 0)           # → 0            missing → default
d.get("x", [])          # → []           missing → default (NEW empty list each call)

# Common idiom — count without KeyError:
counts = {}
for ch in "banana":
    counts[ch] = counts.get(ch, 0) + 1
# → {'b': 1, 'a': 3, 'n': 2}
  • Use .get when "key missing" is a normal case; use d[key] when missing means a bug (let it crash)
  • For counting/grouping, prefer collections.Counter or defaultdict(int) — see counter.html

🪤 Pop template — d.pop(key, default)

Removes and returns the value. With a default, never raises; without, raises KeyError on miss.

d = {"a": 1, "b": 2}

d.pop("a")              # → 1            removed; d is now {"b": 2}
d.pop("x", None)        # → None         missing, default returned (no KeyError)
d.pop("x")              # → KeyError: 'x'  no default, missing key crashes
  • Pass a sentinel default (commonly None) when the key may not exist
  • Returns the VALUE — discard with _ = d.pop(k, None) if you just want to delete

🌱 Setdefault template — d.setdefault(key, default)

Returns existing value OR inserts the default and returns it. Mutates the dict.

d = {"a": 1}

d.setdefault("a", 99)        # → 1     "a" exists, return existing; d unchanged
d.setdefault("b", 99)        # → 99    "b" missing, INSERT 99, return 99; d = {"a":1, "b":99}

# Common idiom — group items into lists per key:
groups = {}
for name, team in people:
    groups.setdefault(team, []).append(name)
# Each missing team gets a fresh [] inserted, then name is appended.
# For this exact pattern, defaultdict(list) is cleaner.
  • Mutating: very different from get, which never inserts
  • For grouping into containers, collections.defaultdict(list) is usually cleaner

🧩 Fromkeys template — dict.fromkeys(iterable, value=None)

Build a dict from a sequence of keys, all mapped to the same value. Two slots: the keys, and the shared value.

dict.fromkeys(["a", "b", "c"])         # → {'a': None, 'b': None, 'c': None}
dict.fromkeys(["a", "b"], 0)          # → {'a': 0, 'b': 0}
dict.fromkeys("abc", 0)             # → {'a': 0, 'b': 0, 'c': 0}     string is iterable

# Dedup-while-preserving-order trick (Python 3.7+):
list(dict.fromkeys([3, 1, 3, 2, 1]))    # → [3, 1, 2]    set() doesn't preserve order

📐 Dict comprehension template — {k: v for k, v in items if cond}

Same shape as a list comprehension, but with two-slot output (key: value).

{ key_expr : val_expr  for k, v in items  if keep_condition }
       ↑          ↑              ↑                      ↑
       key        value          iteration              optional filter
       output     output

# Invert a dict (value → key):
d = {"a": 1, "b": 2}
{v: k for k, v in d.items()}              # → {1: 'a', 2: 'b'}

# Filter: keep only large values:
{k: v for k, v in d.items() if v > 1}      # → {'b': 2}

# Build from two parallel lists:
keys = ["a", "b", "c"]
vals = [1, 2, 3]
{k: v for k, v in zip(keys, vals)}        # → {'a': 1, 'b': 2, 'c': 3}

Same template anatomy as the list comprehension — see list.html for the full slot breakdown.

🔀 Merge template — d1 | d2 · d1 |= d2 · {**d1, **d2}

Three forms — pick by whether you want a NEW dict, in-place mutation, or compatibility with older Python.

d1 = {"a": 1, "b": 2}
d2 = {"b": 99, "c": 3}

# ── 1) d1 | d2  →  NEW dict (Python 3.9+) ──
d1 | d2                  # → {"a": 1, "b": 99, "c": 3}    d2 wins on collision; d1, d2 unchanged

# ── 2) d1 |= d2  →  IN-PLACE merge (Python 3.9+) ──
d1 |= d2                 # mutates d1; d2 wins on collision; d2 unchanged
# d1 is now {"a": 1, "b": 99, "c": 3}
# Equivalent to: d1.update(d2)

# ── 3) {**d1, **d2}  →  NEW dict (works any version) ──
{**d1, **d2}             # same result as d1 | d2

# ── Inline overrides via ** ──
{**d1, "b": 0}            # → {"a": 1, "b": 0}   override specific keys cleanly
  • | returns a NEW dict — d1 and d2 are untouched. Use when you must preserve both inputs
  • |= mutates d1 in place — identical to d1.update(d2). Use when d1 is yours to modify
  • {**d1, **d2} works on any Python version (pre-3.9). Also handy for inline overrides: {**defaults, "key": override}
  • The RIGHT side always wins on key conflicts — same rule for all three forms
  • For pre-3.9 in-place merge, use d1.update(d2) — see the "More dict methods" box below

🌟 Double-star unpacking — **dict (keyword arguments)

**dict opens a dictionary and passes each key: value pair as a keyword argument to a function. Mirror image of **kwargs in a definition.

# Two contexts — mirror images:
# At a CALL site (spread out):
f(**mapping)                 # spreads as keyword args

# In a DEFINITION (collect up):
def f(**kwargs): ...         # collects extra kwargs into a dict

# ── Mental model ──
f(**{"name": "Nirty", "age": 35})
# is identical to:
f(name="Nirty", age=35)

# ── Use case 1: config dict → sorted options ──
options = {"key": lambda x: -x, "reverse": False}
sorted([3, 1, 5, 2], **options)
# → [5, 3, 2, 1]
# same as: sorted([3,1,5,2], key=lambda x: -x, reverse=False)

# ── Use case 2: object construction from a dict ──
class Person:
    def __init__(self, name, age):
        self.name, self.age = name, age

data = {"name": "Nirty", "age": 35}
p = Person(**data)           # same as Person(name="Nirty", age=35)

# ── Use case 3: function wrappers / forward options ──
def run_query(sql, timeout=30, retries=3):
    ...

config = {"timeout": 60, "retries": 5}
run_query("SELECT * FROM users", **config)
# same as: run_query("SELECT * FROM users", timeout=60, retries=5)

# ── Use case 4: library calls ──
args = {"url": "https://example.com", "timeout": 10}
requests.get(**args)         # same as: requests.get(url="...", timeout=10)

# ── Definition-time collection ──
def log(level, **fields):       # fields is a DICT of all extra kwargs
    print(level, fields)
log("INFO", user="a", action="login")
# → INFO {'user': 'a', 'action': 'login'}

⚠️ Gotcha — **dict keys must match parameter names

def add(a, b):
    return a + b

# ✅ Keys match parameter names:
add(**{"a": 10, "b": 20})       # → 30

# ❌ Keys don't match — TypeError:
add(**{"x": 10, "y": 20})
# → TypeError: add() got an unexpected keyword argument 'x'

# ❌ Without ** — passes the whole dict as ONE positional arg:
add({"a": 10, "b": 20})
# → TypeError: add() missing 1 required positional argument: 'b'
  • Dictionary keys MUST match the target function's parameter names exactly — typo in a key = TypeError
  • Without **, f(d) passes the whole dict as one positional argument (almost never what you want)
  • In definitions, def f(**kwargs) collects extra kwargs into a dict — the mirror operation
  • Companion: * for positional/iterable unpacking — see list.html → Syntax Templates

⚠️ Gotcha — d[k] vs d.get(k)

Same mental model as string find vs index: missing-is-normal vs missing-is-a-bug.

d = {"a": 1}

d["a"]              # → 1                    present
d["x"]              # → KeyError: 'x'        missing key crashes
d.get("x")           # → None                 safe sentinel
d.get("x", 0)        # → 0                    safe with custom default

# Use [] when missing means a bug (validate input by letting it crash):
config["required_field"]

# Use .get when missing is expected (counts, optional config, etc.):
counts.get("new_key", 0)

🚨 Gotcha — fromkeys shares the SAME mutable default

All keys point to the SAME object — same shape as the [[0]*cols]*rows bug from lists.

# ❌ TRAP — every key points to the SAME list:
d = dict.fromkeys(["a", "b", "c"], [])
d["a"].append(1)
# d is now {'a': [1], 'b': [1], 'c': [1]}    ← all three lists are the SAME object!

# ✅ Use a comprehension to get fresh containers per key:
d = {k: [] for k in ["a", "b", "c"]}
d["a"].append(1)
# d is {'a': [1], 'b': [], 'c': []}    ← each key got its own list

Same shared-reference bug as list 2D grid init. fromkeys is safe ONLY for immutable defaults (numbers, strings, tuples, None).

See the "More dict methods" tips-box below for popitem, merge operators (|, |=), update, and the full method reference.

d = {}                             # empty dict
d = {"a": 1, "b": 2}             # literal
d = dict(a=1, b=2)               # keyword args
d = {x: x**2 for x in range(5)}  # comprehension
d = dict.fromkeys(["a","b"], 0)   # {"a":0,"b":0}

{} creates an empty dict, NOT a set. Use set() for empty sets.

Operations

d = {"apple": 3, "banana": 5}

# ── Add / Update ──
d["cherry"] = 7             # add new key
d["apple"] = 10              # update existing
d.update({"fig": 2})        # merge another dict

# ── Access ──
d["apple"]                   # 10 (KeyError if missing!)
d.get("grape", 0)            # 0 (safe with default)

# ── Remove ──
del d["banana"]              # remove key
d.pop("cherry")              # remove & return value → 7
d.pop("missing", None)       # safe pop → None

# ── Check ──
"apple" in d                # True (checks keys, O(1))
10 in d.values()            # check value exists

# ── Iterate ──
for k in d:                 # keys only
for v in d.values():        # values only
for k, v in d.items():      # key-value pairs

# ── Size ──
len(d)                      # number of keys

# ── Frequency pattern (plain dict) ──
freq = {}
for ch in "hello":
    freq[ch] = freq.get(ch, 0) + 1
# → {'h':1, 'e':1, 'l':2, 'o':1}

📌 For frequency counting, grouping by key, or multiset arithmetic without manual .get(..., 0) guards, reach for DefaultDict / Counter — Counter("hello") is the one-line equivalent of the loop above, plus you get .most_common(n), +/-/&/| arithmetic, and zero-default lookups for free.

🗝️ More dict methods worth knowing

# ── setdefault — get-or-create in one call ──
d.setdefault(k, default)             # if k not in d: d[k] = default; returns d[k]
# Used for nested-dict trie: see Algorithms → Trie
trie.setdefault(c, {})

# ── pop with default — safe pop, no KeyError ──
d.pop("missing", None)               # returns None instead of raising
removed = d.pop("k", sentinel)        # swap-and-detect-existed pattern

# ── popitem — pops LAST-INSERTED pair (LIFO since 3.7) ──
d.popitem()                          # returns (key, value); raises if empty
# LRU-style cache eviction trick — see Classes → OOD Building Blocks

# ── Dict merge (3.9+) ──
d1 | d2                              # NEW dict; d2 wins on key conflicts
d1 |= d2                             # mutate d1 in place (same as d1.update(d2))
{**d1, **d2}                         # pre-3.9 form (also works for inline overrides)

# ── update — merge into existing dict ──
d.update({"a": 1, "b": 2})           # bulk insert/overwrite
d.update(c=3, d=4)                   # keyword form

# ── Iteration views (live — reflect later mutations) ──
d.keys()                             # view of keys — supports set ops!
d1.keys() & d2.keys()                # keys present in both dicts (intersection)
d1.keys() - d2.keys()                # keys in d1 but not d2

# ── dict.fromkeys — bulk init with same value ──
dict.fromkeys(["a", "b", "c"], 0)   # {"a":0, "b":0, "c":0}
# ⚠️ Don't pass a mutable default — all keys would alias the same object

🛠️ Built-ins commonly paired with dicts

When you write dict code in interviews, these are the built-ins you reach for most.

# ── Iterating ──
for k in d: ...                           # keys (default)
for k, v in d.items(): ...                # pairs — most common
for v in d.values(): ...

# ── argmax/argmin via key= ──
max(d, key=d.get)                         # KEY whose VALUE is largest
min(d, key=d.get)                         # smallest-value key
max(d.items(), key=lambda kv: kv[1])      # full (key, value) pair

# ── Sorting items ──
sorted(d.items(), key=lambda kv: kv[1])   # by value asc
sorted(d, key=d.get, reverse=True)        # keys ordered by value desc

# ── Building from iterables ──
{k: v for k, v in zip(keys, vals)}        # dict comprehension
dict(enumerate(arr))                      # {0: arr[0], 1: arr[1], ...}
dict.fromkeys(arr, 0)                     # unique keys with shared init value

# ── Aggregating ──
sum(d.values())
any(v == 0 for v in d.values())
len(d)                                    # number of keys

Full reference for any of these: Python Builtins.
