Python · built-ins + itertools + functools + math

Functions cheatsheet

lazy returns iterator (consume once) eager returns concrete value/list 3.x+ version-gated O(1) O(log n) O(n) O(n log n)
What these are called: top-level names like max, sorted, sum are built-in functions — always available, no import. The ones that take a function as an argument (key=, map, filter, reduce) are higher-order functions. itertools, functools, and math are stdlib modules — import required, but on every Python install.
Lazy vs eager: lazy functions return an iterator that yields one element at a time and is exhausted after one pass. Wrap in list(...) if you need to iterate twice or index into it.

The key= idiom

The unifying concept: max, min, sorted, and heapq.nlargest/nsmallest all accept a key function. The function is called once per element; its return value is what gets compared. Tuples compare lexicographically, so multi-key sorts fall out for free.

argmax / argmin

max(d, key=d.get)        # key with largest value
min(d, key=d.get)        # key with smallest value
max(range(len(a)), key=a.__getitem__)  # argmax of list

Sort by multiple keys

sorted(emps, key=lambda e:(e.dept, -e.salary, e.name))
# dept ascending, salary descending, name asc

Sort dict by value

sorted(d.items(), key=lambda kv: kv[1])
or
sorted(d.items(), key=operator.itemgetter(1))

Closest to target

min(points, key=lambda p: (p[0]-x)**2 + (p[1]-y)**2)

Stable sort, multi-pass

# Sort by secondary first, then primary — stability preserves order
a.sort(key=lambda x: x.name)       # secondary
a.sort(key=lambda x: x.priority)   # primary

Custom compare (cmp_to_key)

# LC 179: largest number by concatenation
def cmp(a, b):
    return -1 if a+b > b+a else 1
nums = sorted(map(str, nums), key=cmp_to_key(cmp))

Transpose / unzip

rows = [[1,2,3], [4,5,6]]
list(zip(*rows))   # [(1,4),(2,5),(3,6)]

pairs = [(1,'a'),(2,'b')]
nums, chars = zip(*pairs)

Prefix sums / running max

from itertools import accumulate

rmax = list(accumulate(nums, max))       # running max
pre0 = list(accumulate(nums, initial=0)) # pre[0]=0

Pairwise sliding window

from itertools import pairwise
diffs = [b-a for a,b in pairwise(nums)]
ok = all(b >= a for a,b in pairwise(nums))

Cartesian product replaces nested loops

from itertools import product
for r, c in product(range(R), range(C)):
    ...
for mask in product([0,1], repeat=n):
    ...   # enumerate 2^n bitmasks

Generate all subsets / combinations

from itertools import combinations, chain
def powerset(it):
    s = list(it)
    return chain.from_iterable(
        combinations(s, r) for r in range(len(s)+1))

GCD / XOR / OR of array

from functools import reduce
import math, operator
reduce(math.gcd, nums)
reduce(operator.xor, nums)
reduce(operator.or_, masks)

Memoized DP

from functools import cache
@cache
def dp(i, j):
    if i >= n or j >= m: return 0
    return max(dp(i+1, j), dp(i, j+1)) + grid[i][j]

Modular exponentiation

pow(2, 10**9, 10**9 + 7)   # fast modexp
pow(a, -1, p)              # modular inverse, 3.8+

Group consecutive runs

from itertools import groupby
# run-length encode
[(k, sum(1 for _ in g)) for k, g in groupby(s)]
# "aaabbc" → [('a',3),('b',2),('c',1)]

Group by attribute (full)

# groupby needs sorted input for full grouping
items.sort(key=lambda x: x.cat)
for cat, group in groupby(items, key=lambda x: x.cat):
    ...
# or use defaultdict(list) — usually cleaner

Char arithmetic

idx  = ord(c) - ord('a')           # 0..25
char = chr(ord('a') + idx)
counts = [0] * 26
for c in s: counts[ord(c)-ord('a')] += 1

Bitmask tricks

n.bit_length()          # floor(log2)+1
n.bit_count()           # popcount (3.10+)
bin(n).count('1')       # popcount pre-3.10
n & (n - 1)             # clear lowest set bit
n & -n                  # isolate lowest set bit

Ceiling division (no float)

-(-a // b)              # ceil(a/b), all-integer
(a + b - 1) // b        # same, positive only

Find first match

next((x for x in nums if pred(x)), default)
next((i for i,x in enumerate(nums) if pred(x)), -1)

Comprehension equivalents

# map/filter via comprehension — usually clearer
[f(x) for x in it]                    # map(f, it)
[x for x in it if pred(x)]            # filter(pred, it)
{f(x): g(x) for x in it}              # dict from items
{x for x in it}                       # set
(f(x) for x in it)                    # generator (lazy)

Gotchas

  • map, filter, zip, and most itertools return iterators that are consumed once. Wrap in list(...) if you need to reuse or print multiple times.
  • max([], default=...) / min([], default=...) — without default, empty iterables raise ValueError.
  • reduce(f, []) without an initializer raises TypeError. Always provide init when the iterable might be empty.
  • round uses banker's rounding: round(0.5) == 0, round(1.5) == 2, round(2.5) == 2. Not what most people expect.
  • sum([]) returns 0; sum with a non-numeric start is rejected (don't use it to flatten lists — use chain.from_iterable or a comprehension).
  • itertools.groupby only groups consecutive equal-key elements. Sort first, or use defaultdict(list) for full grouping.
  • itertools.count, cycle, repeat (no times) are infinite. Combining with list(...) hangs forever.
  • zip stops at the shortest iterable silently. Use strict=True (3.10+) when lengths must match, or zip_longest for padding.
  • sorted is stable — equal keys preserve original order. Use this for multi-pass sorts: sort by secondary first, then primary.
  • The key= function is called once per element, not per comparison (Schwartzian transform internally). Heavy key functions are fine.
  • list.sort() sorts in place and returns None; sorted(list) returns a new list. Don't write a = a.sort().
  • functools.reduce is imported from functools, not a builtin (since Python 3). Watch for "NameError: name 'reduce' is not defined".
  • math.log2(n) returns a float — fine for size, but for exact integer log use n.bit_length() - 1.
  • pow(b, e, m) is dramatically faster than (b**e) % m for large e — and avoids creating huge intermediates.
  • int(s) parses base 10 by default; int(s, 0) interprets 0b/0o/0x prefixes; int(s, base) for explicit base.
  • any and all short-circuit, so generator expressions inside them stop as early as possible — pass a generator, not a list.
  • @cache requires hashable arguments. Lists/sets/dicts as args won't work — convert to tuple/frozenset/tuple(items()) first.
